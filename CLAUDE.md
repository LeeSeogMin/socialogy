# 사회통계 강의자료 프로젝트

## 프로젝트 개요

이 프로젝트는 **사회통계 강의자료 자동 제작 시스템**입니다.

### 강의 정보
- **과목명**: 사회통계(사회조사·통계분석 기초 + 프로젝트, GitHub Copilot 활용)
- **대상**: 학부생 (사회조사방법론 1 이수 권장)
- **구성**: 14주 × 주 2회 = 28회차 (회당 90분)
- **운영**: 10주 완주(핵심 분석) + 4주 팀프로젝트(3인)
- **최종 목차**: `contents.md` 참조

### 과목 핵심
사회학 연구에서 자주 쓰는 통계적 사고와 기초 분석기법을 실습으로 익히며, GitHub Copilot을 **코딩·디버깅·해석문 다듬기 조교**로 활용한다. 최종 결론은 **코드 실행 결과와 근거**에 기반해 학생이 판단한다.

## 최우선 원칙

### 1. 수업 내 완결형 과제 (핵심)
- 과제는 **30–40분 내 완료** 가능해야 한다
- 3단계 구성: (1) 실행 → (2) 확인 → (3) 해석 (각 10–15분)
- 기본(Basic: 필수) + 확장(Plus: 가산점) 두 트랙 제공

### 2. 표준 제출물 형식 (A–E)
모든 과제는 동일한 구조로 제출한다:
- (A) 분석 질문(1–2문장)
- (B) 사용 변수/수준(명목·서열·등간/비율) 표기
- (C) 결과 근거(핵심 표/그래프 1–2개, 또는 수치 3개 이내)
- (D) 해석문(5–7문장): "무엇을 비교했고 → 무엇을 봤고 → 그래서 무엇을 말할 수 있고/없는가"
- (E) Copilot 사용 요약(2–3줄): 어떤 요청을 했고 무엇을 검증했는가

### 3. Copilot 사용 정책
- **허용**: 개념 설명, 분석 절차 안내, 디버깅, 결과 해석 초안, 글쓰기 도움
- **금지**: 개인정보/민감정보 업로드, 시험 답안 복사 제출, 데이터 조작/위조, 타인 과제 복제
- **증빙**: 과제에 결과 근거 + Copilot 사용 요약 첨부
- **책임**: Copilot 답변은 참고이며, 최종 결과의 정확성은 학생 책임

### 4. 강의자료 집필 방향

**핵심 원칙**: 학부생이 통계적 사고의 원리를 이해하고 실습으로 적용할 수 있도록 하는 것이 목표.

#### 4.1 단계적 설명 원칙
- **"왜 필요한가" → "핵심 개념" → "코드로 실행" → "결과 해석"** 순서
- 수학적 증명보다는 **직관적 이해**에 집중
- Copilot 프롬프트를 미리 제시하여 학생이 헤매지 않게 안내

#### 4.2 코드 제시 원칙
- **핵심 코드만 본문에 포함** (5–10줄)
- **전체 구현 코드는 별도 파일로 분리**: `practice/W{주차}/code/`
- 한국어 주석으로 각 단계의 의도 설명 ("왜" 중심)

#### 4.3 실행 결과와 해석 필수
- 실제 실행 결과만 사용 (가상 결과 금지)
- 숫자만 나열하지 않고 의미 해석
- 해석문 템플릿 제공: "(분석목적) → (핵심근거) → (결론) → (주의/한계)"

#### 4.4 회차별 필수 구성요소
각 회차 자료는 다음을 반드시 포함한다:
- 오늘의 질문(1문장)
- 목표(3개 이내)
- 핵심 개념(3개 이내, 정의/가정/자주하는 오해)
- 예시(표/그래프 1개)
- 실습 과제(3단계 + 체크리스트, 기본/확장 트랙)
- Copilot 프롬프트(필수 2개 + 선택 1개)
- 상호작용 장치(2개 이상: 예측 투표, 오답 수집, 짝 점검, 미니 디버깅)
- 흔한 오류 3개 + 확인법
- Exit ticket(1문항)

## 회차 문서 통합 형식 (90분 구조)

각 회차는 **하나의 문서**로 강의자료와 과제를 통합한다.

### 문서 구조

```
# W{주차}{A/B}. 제목
> 오늘의 질문 (1문장)

학습 목표 (3개 이내)

# (0–10분) 오늘의 질문/목표/진단
  빠른 진단 1문항

# (10–30분) 미니 강의
  핵심 개념 3개 이내 + 예시 1개

# (30–70분) 실습 과제 (수업 내 완결형)
  3단계 (실행→확인→해석)
  Copilot 프롬프트

# (70–85분) 정리/오류 패턴 공유 + 비판적 읽기
  해석문 "모범 구조" 제시 + 기존 연구 표/결과 해석 판단 연습

# (85–90분) Exit ticket
  1문항 + 다음 시간 예고
```

### 서식 세부 규칙
- **표/그림 제목**: 표 위에 `**표 W{주차}.{순번}** 제목`, 그림 아래에 `**그림 W{주차}.{순번}** 제목`
- **과제 단계 번호**: 원문자 사용 — **① ② ③**
- **번호 체계**: `W{주차}.{순번}` (예: 표 W05.1, 그림 W05.2)

```

        ▼

        │
        ▼

        ▼
```

## 핵심 작업 규칙

### 문체 가이드
- **문체**: 명확하고 논리적인 설명체, 학생이 이해하기 쉽고 가독성 높게 서술
- **종결어미**: 격식체 평서문 ('이다', '한다', '보인다')
- **용어**: 전문 용어 첫 등장 시 **굵게** + 영문 병기 (예: "**카이자승 검정**(Chi-square test)")
- **난이도 조절**: 어려운 개념은 "쉽게 말해서..." 또는 비유로 보충 설명
- **개조식 허용**: 불릿 포인트와 번호 목록을 자유롭게 사용하되 각 항목에 충분한 설명 포함
- **해석문 템플릿** 제공:
  - "(분석목적) ___를 보기 위해 ___를 사용했다."
  - "(핵심근거) 출력에서 ___(검정통계량/df/p값)을 확인했다."
  - "(결론) 유의수준 ___에서 ___(독립/차이/관계)를 ___(기각/채택)한다."
  - "(주의) 이는 ___를 의미하지 않으며, ___의 한계가 있다."

### 코드 스타일
- Python 3.10+ (또는 R 대체 가능)
- 실행 환경: Jupyter 또는 Google Colab
- 한국어 주석/docstring
- 실제 실행 결과만 사용 (더미/Mock 결과 금지)

### 실습 환경
- 코드/분석: Python(pandas, scipy, statsmodels) 또는 R
- AI 도우미: GitHub Copilot
- B회차 조별 과제: 같은 조(2–3인)끼리 협의·수행, 조별 1부 제출
