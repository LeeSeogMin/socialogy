# AI 시대형 강의계획서/강의자료 제작 지침(Contents Guide)

## 0) 적용 범위(고정 조건)
- 수업 형태: 주 2회, 회당 1시간 30분(90분)
- 진행 방식: **강의자(교수) 강의 + 학생 자율 실습/과제**
- 과제 원칙: **수업시간 내 전원 완료 가능**(과제는 “수업 내 완결형”으로 설계)
- 실습 도구: GitHub Copilot을 활용(코드 초안, 디버깅 절차, 해석문 다듬기) + 오픈 도구(Python/R, Jupyter/Colab)
- 기존 주제 유지: 자료 부호화·요약통계·카이자승·가설검정·t검정/ANOVA·회귀(단순/중다) 등

## 1) 강의계획서(syllabus.md) 작성 원칙
### 1-1. 과목 정체성 문장(첫 페이지에 반드시 포함)
- “본 과목은 사회학 연구에서 자주 쓰는 통계적 사고와 기초 분석기법을 실습으로 익히며, 학습 과정에서 **GitHub Copilot을 조교(튜터)로 활용**하여 코딩·오류 수정·결과 해석의 언어화를 훈련한다.”

### 1-2. 학습성과(측정 가능한 문장으로)
학기 말 학생이 할 수 있어야 하는 것을 **행동동사**로 명시한다(예: 입력/검증/분석/해석/보고).
- (데이터 준비) 설문 데이터를 **부호화·정제**하고 변수 수준(명목/서열/등간·비율)을 **구분**한다.
- (분석 실행) 코드/노트북에서 핵심 분석(요약·분할표·검정·회귀)을 **실행**하고 산출물을 **정리**한다.
- (해석/보고) 결과를 “질문–가설–검정–결론–한계” 구조로 **서술**한다.
- (AI 활용) Copilot을 사용해 절차를 **점검**하되, 최종 판단(가설 채택/기각, 해석 문장)은 근거(코드 실행 결과·표·그래프·p값 등)를 들어 **자기 언어로 작성**한다.

### 1-3. Copilot 활용 정책(명확히 적어 혼선을 줄이기)
강의계획서에 아래 4가지를 고정 문구로 포함한다.
- 허용: 개념 설명 요청, 분석 절차 안내, 디버깅 절차, 결과 해석 초안, 글쓰기(보고서 문장 다듬기) 도움
- 금지: 개인정보/민감정보 업로드, 시험·퀴즈 답안 “그대로 제출”, 데이터 조작/위조, 타인 과제 복제
- 증빙: 과제 제출물에는 **(1) 결과 근거(표/그래프/수치)**와 **(2) Copilot 사용 요약(2–3줄)**을 함께 첨부
- 책임: Copilot 답변은 참고이며, **최종 결과의 정확성은 학생 책임**(교수자는 검증 방법을 평가)

### 1-4. 평가 설계(“수업 내 완결형”에 맞게)
- 과제는 “긴 리포트”보다 **짧은 산출물(표/해석문/체크리스트)** 중심으로 구성한다.
- 과제 채점 기준은 “정답”보다 **근거 제시(출력표) + 해석의 논리성 + 재현 가능성**에 둔다.
- 평가는 **중간·기말 + 매주 과제**로 구성한다(이 3가지가 기본 축).
- 비율은 강의 운영에 따라 정하되, “매주 수행”을 반영해 **주간 과제 비중을 가장 크게** 두는 편이 운영이 안정적이다.
  - 권장 예시 A: 중간 30% / 기말 30% / 매주 과제 40%
  - 권장 예시 B: 중간 25% / 기말 25% / 매주 과제 50%

### 1-5. 중간·기말 시험 설계 지침(객관식+주관식 혼합)
시험은 “계산”보다 **선택의 타당성 + 출력 해석 + 문장화**를 평가하도록 설계한다.
- 시험 시간: 90분(수업 1회차 기준)
- 권장 배점 비율: 객관식 60% + 주관식 40%
  - 객관식(예): 개념/가정/용어, 척도 구분(명목·서열·등간/비율), 어떤 검정을 선택해야 하는지, p값/유의수준/자유도 해석 포인트
  - 주관식(예): 결과표/그래프/짧은 코드 조각을 제시하고 ① 분석/검정 선택 이유 ② 핵심 수치/패턴 근거 ③ 결론 ④ “이 결과가 의미하지 않는 것(인과 아님 등)”을 포함한 해석문(5–7문장)
- AI 정책(권장): **시험 중 Copilot/인터넷 사용 금지**로 명시(분쟁 예방). 필요 시 오픈북 여부만 별도로 선택한다.

### 1-6. 매주 과제 평가·환류 지침(반드시 ‘되돌려주기’ 포함)
매주 과제는 점수화하고, 학생이 “다음 과제에 바로 반영”할 수 있도록 **빠른 피드백**을 운영 규칙으로 고정한다.
- 과제 완료: 수업시간 내 완결(§4 기준 유지)
- 반환 시점(SLA): **다음 주 첫 수업 시작 전/또는 첫 10분 내** 반환을 원칙으로 한다.
- 반환 방식: 점수 + 루브릭 체크 + 1개 코멘트(가장 중요한 개선점 1개만)
- 표준 루브릭(권장 4항목, 각 0–2점 또는 0–3점):
  1) 분석 선택의 타당성(척도/가정 포함)
  2) 근거 제시(표/그래프/수치의 핵심 근거)
  3) 해석의 정확성(비인과 표현, 한계 언급)
  4) 재현 가능성(절차 기록 + Copilot 사용 요약의 적절성)

## 2) 주차 구성 원칙(주 2회 × 14주 → 28회차)
### 2-1. 단원을 통합하고 ‘프로젝트 스튜디오’ 시간을 확보한다
각 주차(Week N)는 **A회차(통합 개념)** + **B회차(스튜디오/체크포인트)**로 구성한다.
- 원칙: B회차의 45분 이상을 **프로젝트 작업 시간**으로 고정(수업 내 완결형 체크포인트).
- 효과: “작은 주제 나열”을 줄이고, 학생이 분석 흐름을 **끝까지 완주**하도록 설계한다.

### 2-3. 10주 완주 + 4주 팀프로젝트(3인)로 설계한다(권장 운영안)
- 1–10주: 핵심 분석 흐름을 모두 경험(데이터 준비→요약→관계/검정→회귀→해석/보고)
- 11–14주: **3인 팀프로젝트 스튜디오**(체크포인트 3회 + 쇼케이스 1회)
- 주간 과제 정의:
  - 1–10주: 개인 과제(기법 습득)
  - 11–14주: 팀 체크포인트(팀 제출) + 팀원별 기여 3줄(무임승차 방지)

### 2-4. 주차별 설계 시 유의점(전환·연결·선행)
- **데이터 리터러시 내장(2주)**: 데이터 준비 주차에서 코딩/결측 처리뿐 아니라, **"이 데이터는 누가, 왜, 어떻게 모았는가?"**를 반드시 짚는다. 데이터의 출처·맥락·한계를 의심하는 태도가 이후 해석문의 "(주의) 한계" 작성에 직결된다.
- **기술→추론 전환 브릿지(4→5주)**: 3–4주(기술통계/카이자승)에서 5주(가설검정 프레임)로 넘어갈 때, **"기술통계(있는 그대로 요약)와 추론통계(모집단에 대해 말하기)의 차이"**를 명시적으로 짚는 브릿지 시간을 A회차 도입부에 배치한다.
- **10주 통합 정리 — 미니 완주 프로젝트**: 10주차는 단순 복습이 아니라, **하나의 연구 질문을 처음부터 끝까지(데이터 선택 → 기술통계 → 검정 선택 → 실행 → 해석문) 완주하는 "미니 프로젝트"**로 설계한다. 이 경험이 11주 팀프로젝트로 자연스럽게 이어진다.
- **팀프로젝트 데이터 선행 조사(9–10주)**: 11주 팀 킥오프에서 데이터를 처음 고르면 시간이 부족하다. **9주 또는 10주 과제에 "팀프로젝트에 쓸 공개 데이터 후보 3개 + 각각의 변수/척도/한계 메모"를 선행 과제로 포함**하여, 11주 킥오프에서 바로 분석 계획에 착수할 수 있게 한다.

### 2-2. “한 회차에 하나의 핵심 질문(Driving Question)” — 질문 중심 설계
회차 자료 첫 줄에 반드시 1문장 **연구 질문**을 둔다. 주차 제목도 기법 이름이 아니라 질문으로 표기한다.
- 기법 중심(X): “카이자승 검정” → 질문 중심(O): **”두 범주형 변수가 관련 있는가?”**
- 학생이 “어떤 질문에 어떤 도구를 쓰는가”를 사고 흐름으로 익히게 하는 것이 목표이다.
- 기법은 질문에 답하기 위한 **도구**로 등장한다. 도구 자체가 주인공이 아니다.

## 3) 90분 수업(회차) 표준 운영 템플릿
회차별 강의자료는 아래 시간표를 기본으로 만든다(±5분 조정 가능).
1. (0–10분) **오늘의 질문/목표/평가 기준** 공유 + 빠른 진단(1문항)
2. (10–30분) **미니 강의**(핵심 개념 3개 이내, 예시 1개)
3. (30–70분) **자율 실습 과제(수업 내 완결형)**  
   - 개인→짝/조 점검→전체 공유 순으로
4. (70–85분) **정리/오류 패턴 공유** + 해석문 “모범 구조” 제시 + **비판적 읽기**(5–10분: 기존 연구의 표/결과를 보고 “이 해석이 적절한가?” 판단 연습)
5. (85–90분) **Exit ticket**(1문항 + 다음 시간 예고)

## 4) “수업 내 완결형 과제” 설계 지침(핵심)
### 4-1. 과제는 30–40분 내 완료 가능해야 한다
- 과제는 **3단계**로 쪼갠다: (1) 실행 (2) 확인 (3) 해석
- 각 단계는 **10–15분** 목표 시간과 “완료 체크”를 제공한다.

### 4-2. 과제 산출물(제출 포맷)을 표준화한다
모든 과제는 동일한 제출 구조를 사용한다(학생 부담↓, 채점 효율↑).
- (A) 분석 질문(1–2문장)
- (B) 사용 변수/수준(명목·서열·등간/비율) 표기
- (C) 결과 근거(핵심 표/그래프 1–2개, 또는 수치 3개 이내)
- (D) 해석문(5–7문장): “무엇을 비교했고 → 무엇을 봤고 → 그래서 무엇을 말할 수 있고/없는가”
- (E) Copilot 사용 요약(2–3줄): 어떤 요청을 했고 무엇을 검증했는가

### 4-3. 난이도 조절 장치(완주율 보장)
과제마다 “기본/확장” 두 트랙을 둔다.
- 기본(Basic): 전원 필수(채점 대상)
- 확장(Plus): 빠른 학생용(가산점 또는 선택)

### 4-4. 팀 체크포인트 과제(11–14주) 설계 지침
팀프로젝트는 “마지막에 몰아서”가 아니라, 주간 체크포인트로 쪼개어 수업시간 내 진도를 보장한다.
- 팀 규모: 3명
- 체크포인트(권장 4회):
  1) 질문/가설/분석계획(변수·척도·대상·한계 포함)
  2) 핵심 결과 2개 + 해석 초안(근거 포함)
  3) 최종 산출물 리허설(재현성 체크 포함)
  4) 쇼케이스 + 최종 제출
- 제출 형식: 기존 A–E 포맷을 팀 기준으로 유지하되, 맨 아래에 “팀원별 기여 3줄”을 추가한다.

## 5) Copilot 활용 방식(실습에서 실제로 쓰게 만드는 규칙)
### 5-1. Copilot 역할을 ‘조교’로 고정한다
회차 자료에 “Copilot에게 물어볼 질문”을 **미리 제시**한다(학생이 헤매지 않게).
- 예시 프롬프트(공통):
  - “내 변수 A와 B는 각각 어떤 척도(명목/서열/등간·비율)로 보는 게 적절할까? 이유도 말해줘.”
  - “카이자승 검정 결과를 해석할 때 반드시 언급해야 하는 요소 3가지는?”
  - “이 결과표/출력에서 p값, 자유도, 검정통계량(또는 계수)이 어디에 있는지 설명해줘.”
  - “내 해석문이 ‘인과’처럼 읽히지 않게 문장을 고쳐줘(상관/관계 수준으로).”

### 5-2. Copilot 답변을 ‘검증’하도록 과제에 포함한다
과제 문항에 검증 항목을 반드시 넣는다.
- “Copilot의 제안 중 1가지를 **채택/거절**하고, 그 이유를 결과 근거(표/그래프/수치)로 2문장으로 적어라.”

### 5-3. Copilot 사용 안내(강의자료 공통)
- 1회차에 설치/로그인을 안내하고, 이후 과제에서는 “무엇을 요청했고 무엇을 검증했는지”를 짧게 기록하게 한다.
- 개인정보 주의: 실습 데이터는 **익명화된 공개/수업용 데이터만** 사용(학생 개인자료 업로드 금지)

## 6) 교수–학생 상호작용이 가능한 강의자료(슬라이드/핸드아웃) 제작 지침
### 6-1. 한 회차 자료의 고정 구성(권장)
1) 오늘의 질문(1문장)  
2) 목표(3개 이내)  
3) 핵심 개념(정의/가정/자주하는 오해)  
4) 예시(표/그래프 1개)  
5) 실습 과제(3단계 + 체크리스트)  
6) Copilot 프롬프트(필수 2개 + 선택 1개)  
7) 공유 질문(토론 2개)  
8) Exit ticket(1문항)  

### 6-2. 상호작용 장치(매 회차 최소 2개 포함)

### 2주: 이 데이터를 믿을 수 있는가?

#### A회차 — 데이터 리터러시 + 코딩/결측/변수 정의

**Driving Question**: “이 데이터는 누가, 왜, 어떻게 모았으며, 어떤 한계가 있을까?”

**핵심 개념 (3개)**
1. **데이터 리터러시**: 데이터의 출처·수집 목적·표본 특성·편향 가능성을 파악하는 태도. 수치를 분석하기 전에 “이 데이터를 믿을 수 있는가?”를 먼저 질문한다.
2. **변수의 척도 수준**: 명목(nominal) — 이름만, 서열(ordinal) — 순서 있음, 등간(interval) — 간격 동일, 비율(ratio) — 절대 영점. 척도가 분석 기법 선택을 결정한다.
3. **결측치(missing data)**: 완전 무작위 결측(MCAR) / 무작위 결측(MAR) / 비무작위 결측(MNAR). 결측 유형에 따라 제거·대체·모델 기반 처리를 선택한다.

**집필 방향**
- **데이터 리터러시를 첫 15분(미니 강의)의 핵심으로** — “이 설문은 대학생만 대상이므로, 결과를 전체 국민에게 일반화할 수 없다”와 같은 구체적 사례로 시작
- 척도 구분은 **의사결정 트리 형태**로 제시: “순서가 있는가? → 간격이 동일한가? → 절대 영점이 있는가?”
- 코딩(부호화) 설명 시 실제 설문지 → 코드북 → CSV 변환 흐름을 보여준다
- **흔한 오해**: “리커트 5점 척도는 등간이다” → 사실은 서열로 보는 것이 보수적

**강화 항목**
- ✅ “데이터 신뢰 체크리스트” 제공:
  - [ ] 수집 주체와 목적이 명확한가?
  - [ ] 표본 추출 방법이 적절한가?
  - [ ] 응답률과 결측 패턴이 보고되었는가?
  - [ ] 변수 정의(코드북)가 제공되었는가?
- ✅ 척도 구분 의사결정 트리 (표/그림)
- ✅ 결측치 유형별 처리 전략 비교표

**Copilot 프롬프트 예시**
1. “이 데이터의 변수 '교육수준'은 명목/서열/등간/비율 중 어디에 해당할까? 이유도 설명해줘.”
2. “결측치가 20% 이상인 변수를 어떻게 처리해야 할까? 장단점을 비교해줘.”
3. (선택) “이 데이터의 표본 특성(연령, 성별 분포)이 모집단 대표성을 가지는지 판단하는 방법을 알려줘.”

**참고문헌 방향**
- 홍두승, 설동훈 (2012). 사회조사분석. — 변수 정의/코딩 부분
- Babbie (2021). 사회조사방법론. — 측정 수준 부분

### 3주: 데이터의 전체 모습을 어떻게 파악하고 표현할까?

#### A회차 — 요약·분포·산포 + 해석문 구조

**Driving Question**: “데이터의 전체적인 모습을 어떻게 한눈에 파악하고, 말로 표현할 수 있을까?”

**핵심 개념 (3개)**
1. **중심 경향(central tendency)**: 평균(mean), 중앙값(median), 최빈값(mode). 분포가 치우쳐 있으면 중앙값이 더 대표적이다.
2. **산포(dispersion)**: 범위(range), 분산(variance), 표준편차(std). “평균만으로는 분포의 퍼짐 정도를 알 수 없다.”
3. **해석문 구조**: (분석목적) → (핵심근거) → (결론) → (주의/한계). 이 4단계 틀을 처음 도입하고 이후 모든 과제에서 반복 사용한다.

**집필 방향**
- 평균과 중앙값의 차이를 **소득 분포 예시**로 설명: “소수의 고소득자가 평균을 끌어올린다”
- 시각화는 **히스토그램 + 박스플롯** 두 가지에 집중 (너무 많은 그래프 유형을 나열하지 않는다)
- **해석문 구조를 이 주차에서 처음 소개**하고, 이후 매주 반복 → 습관화
- 기술통계의 한계: “있는 그대로 요약”일 뿐, “모집단에 대한 추론”은 아님을 명시 → 5주 가설검정 전환 브릿지의 씨앗

**강화 항목**
- ✅ “요약통계 해석 체크리스트”: 평균 vs 중앙값 차이가 크면 치우침 의심, 표준편차가 평균에 비해 크면 이질적 집단
- ✅ 그래프 선택 가이드: 연속 → 히스토그램/박스플롯, 범주 → 막대그래프, 관계 → 산점도
- ✅ 해석문 모범/오류 예시 비교 (O/X 형식)

**Copilot 프롬프트 예시**
1. “이 변수의 평균과 중앙값이 크게 다른데, 이유가 뭘까? 분포 형태와 연결해서 설명해줘.”
2. “이 히스토그램을 보고 '정규분포에 가깝다/아니다'를 판단하는 기준을 알려줘.”
3. (선택) “이 기술통계 결과를 사회과학 논문 스타일 해석문(5문장)으로 써줘.”

**참고문헌 방향**
- Babbie (2021) — 기술통계 부분
- OpenIntro Statistics — Ch.1-2 (무료 온라인)

### 4주: 두 범주형 변수가 관련 있는가?

#### A회차 — 분할표·카이자승(가정/해석)

**Driving Question**: “두 범주형 변수의 관계가 우연인지(독립인지) 어떻게 판단할까?”

**핵심 개념 (3개)**
1. **분할표(contingency table/crosstab)**: 두 범주형 변수의 빈도를 행×열로 정리한 표. 관찰빈도 vs 기대빈도를 비교한다.
2. **카이자승 검정(Chi-square test)**: 관찰빈도와 기대빈도의 차이가 우연으로 설명되는지 검정. **가정**: (1) 독립 표본, (2) 기대빈도 ≥ 5.
3. **독립/연관**: “두 변수가 독립이다” = 한 변수의 분포가 다른 변수의 범주와 무관. 연관이 있다 ≠ 인과관계.

**집필 방향**
- **기대빈도의 직관적 설명**: “만약 성별과 지지정당이 완전히 무관하다면, 각 칸의 빈도는 얼마일까?” → 이것이 기대빈도
- 검정통계량(χ²)의 계산 과정을 **1개의 구체적 예제**로 보여주되, 공식 암기보다 **”관찰과 기대의 차이가 클수록 χ²가 커진다”**는 직관을 강조
- **가정 위반 시 대안**: 기대빈도 < 5 → Fisher 정확 검정
- **효과 크기**: Cramér's V를 함께 보고하도록 안내 (유의하더라도 효과가 작을 수 있음)

**강화 항목**
- ✅ 분할표 읽기 연습: “이 표에서 가장 눈에 띄는 패턴은?”
- ✅ p값의 의미: “두 변수가 독립이라는 가정 하에, 이 정도 차이가 우연히 나올 확률”
- ✅ 흔한 오해: “p값이 작다 = 효과가 크다” (X) → Cramér's V로 확인

**Copilot 프롬프트 예시**
1. “카이자승 검정 결과를 해석할 때 반드시 언급해야 하는 요소 3가지는?”
2. “기대빈도가 5 미만인 셀이 있으면 어떻게 해야 하는지 설명해줘.”
3. (선택) “이 분할표 결과가 '인과'처럼 읽히지 않게 해석문을 고쳐줘.”

**흔한 오류**
1. 기대빈도 확인을 건너뜀 → `scipy.stats.chi2_contingency`의 `expected_freq` 출력 확인
2. “유의하다 = 중요하다”로 해석 → 효과 크기(Cramér's V) 함께 보고
3. “카이자승 = 인과” → 반드시 “연관/관계” 표현 사용

**실습 코드 방향**
```python
import pandas as pd
from scipy.stats import chi2_contingency
ct = pd.crosstab(df[“성별”], df[“지지정당”])
chi2, p, dof, expected = chi2_contingency(ct)
print(f”χ² = {chi2:.3f}, df = {dof}, p = {p:.4f}”)
```

### 5주: 표본의 패턴이 모집단에도 존재하는가?

#### A회차 — 기술→추론 전환 + 가설검정 프레임

**Driving Question**: “표본에서 발견한 패턴이 모집단에도 존재한다고 말할 수 있을까?”

**핵심 개념 (3개)**
1. **기술통계 vs 추론통계 전환**: 기술통계 = “이 데이터에서 무엇이 보이는가” (있는 그대로). 추론통계 = “이 패턴이 모집단에서도 성립하는가” (불확실성 하의 판단). **이 전환이 통계학의 핵심 도약이다.**
2. **가설검정 프레임**: 귀무가설(H₀: 차이/관계 없다) vs 대립가설(H₁: 있다) → 검정통계량 계산 → p값 확인 → 판단. “유죄 추정이 아니라, 무죄 추정에서 출발한다”는 법정 비유.
3. **p값과 유의수준(α)**: p값 = “H₀가 참이라면, 이 정도 결과가 우연히 나올 확률”. α(보통 0.05) = “이 확률 이하면 우연이 아니라고 판단하겠다”는 기준선. **p값 < α이면 H₀ 기각.**

**집필 방향**
- **도입부 10분을 기술→추론 전환 브릿지에 할당**: “3–4주차에서 분포를 요약하고 분할표를 만들었다. 하지만 그것은 '이 200명의 표본'에 대한 이야기였다. 이제 '전체 모집단'에 대해 말하려면 무엇이 필요한가?”
- p값 설명에 **동전 던지기 비유**: “동전을 10번 던져 8번 앞면 → 편향된 동전인가, 우연인가?”
- **1종 오류 / 2종 오류**를 간략히 소개 (이름·정의만, 수식 없이)
- **가설검정의 한계**: “유의하다 ≠ 중요하다”, “유의하지 않다 ≠ 효과 없다”

**강화 항목**
- ✅ 가설검정 의사결정 플로우차트: “연구 질문 → 변수 척도 확인 → 검정 선택 → 가설 설정 → 실행 → 판단”
- ✅ p값 오해 TOP 3:
  1. “p = 0.03이면 H₀가 참일 확률이 3%이다” (X)
  2. “p값이 작을수록 효과가 크다” (X)
  3. “p > 0.05이면 H₀가 참이다” (X)
- ✅ 검정 선택 가이드표 (이 주차에서 처음 제시, 이후 반복 참조):

| 연구 질문 | 독립변수 척도 | 종속변수 척도 | 검정 |
|---|---|---|---|
| 범주 간 관계 | 명목 | 명목 | 카이자승 |
| 두 집단 평균 차이 | 명목(2) | 등간/비율 | 독립표본 t-검정 |
| 세 집단 이상 평균 차이 | 명목(3+) | 등간/비율 | ANOVA |
| 두 연속 변수 관계 | 등간/비율 | 등간/비율 | 상관/회귀 |

**Copilot 프롬프트 예시**
1. “p값이 0.03이라는 것이 정확히 무엇을 의미하는지, 흔한 오해 3가지와 함께 설명해줘.”
2. “내 연구 질문이 '___'이고 변수가 ___일 때, 어떤 검정을 써야 하는지 알려줘.”
3. (선택) “1종 오류와 2종 오류의 차이를 일상생활 예시로 설명해줘.”

### 6주: 집단 간 평균이 '진짜' 다른가?

#### A회차 — t-검정/ANOVA(가정/해석)

**Driving Question**: “두 집단(또는 세 집단 이상)의 평균이 '진짜' 다른지 어떻게 판단할까?”

**핵심 개념 (3개)**
1. **독립표본 t-검정(independent samples t-test)**: 두 독립 집단의 평균 차이를 검정. **가정**: (1) 독립성, (2) 정규성, (3) 등분산(Levene 검정). 등분산 위반 시 Welch's t 사용.
2. **ANOVA(Analysis of Variance)**: 세 집단 이상의 평균 차이를 한꺼번에 검정. “왜 t-검정을 3번 하면 안 되는가?” → 다중비교 오류(1종 오류 누적).
3. **사후검정(post-hoc test)**: ANOVA에서 유의한 차이가 나왔을 때, **어느 집단 쌍이 다른지** 확인 (Tukey HSD, Bonferroni).

**집필 방향**
- t-검정 설명 시 **”표본 평균의 차이 ÷ 표준오차”**라는 직관을 강조 — 분자(차이)가 크거나, 분모(변동)가 작으면 유의
- 가정 확인 순서를 **체크리스트**로 제공: 독립성 → 정규성(Shapiro-Wilk) → 등분산(Levene)
- ANOVA는 “여러 집단의 평균이 **모두 같다**는 귀무가설을 검정”이라는 틀로 설명
- 효과 크기: Cohen's d(t-검정), η²(ANOVA)를 함께 보고하도록 안내

**강화 항목**
- ✅ 가정 위반 시 대안 표:
  | 가정 위반 | 대안 |
  |---|---|
  | 정규성 위반 | Mann-Whitney U (t 대안), Kruskal-Wallis (ANOVA 대안) |
  | 등분산 위반 | Welch's t-검정 |
  | 다중비교 | Tukey HSD, Bonferroni 보정 |

**Copilot 프롬프트 예시**
1. “Levene 검정 결과 p=0.02이다. 등분산 가정이 충족되는가? 아니라면 어떻게 해야 하는가?”
2. “ANOVA F값이 유의한데, 어떤 집단이 다른지 어떻게 알 수 있는지 설명해줘.”
3. (선택) “Cohen's d = 0.3이면 효과가 큰 편인가 작은 편인가?”

**흔한 오류**
1. 등분산 가정을 확인하지 않고 t-검정 실행 → `scipy.stats.levene` 먼저 실행
2. ANOVA 유의 → “모든 집단이 다 다르다”로 해석 → 사후검정으로 **어느 쌍**인지 확인
3. 표본 크기가 작은데 정규성을 무시 → n < 30이면 Shapiro-Wilk 필수

**실습 코드 방향**
```python
from scipy import stats
# 독립표본 t-검정
t_stat, p_val = stats.ttest_ind(group_a, group_b)
# 등분산 확인
levene_stat, levene_p = stats.levene(group_a, group_b)
# ANOVA
f_stat, p_val = stats.f_oneway(group1, group2, group3)
```

### 7주: 한 변수가 다른 변수를 얼마나 예측하는가?

#### A회차 — 단순회귀(계수/적합도/해석, 인과 오해 방지)

**Driving Question**: “한 변수가 다른 변수를 얼마나 '예측'하는가? (주의: 예측 ≠ 원인)”

**핵심 개념 (3개)**
1. **단순회귀(simple linear regression)**: Y = β₀ + β₁X + ε. β₁ = “X가 1단위 증가할 때 Y의 **평균 변화량**”. β₀ = X=0일 때 Y의 기대값.
2. **적합도(R²)**: “모델이 Y의 변동을 몇 %나 설명하는가?” R² = 0.25면 25% 설명. **주의**: R²가 높아도 인과가 아니며, R²가 낮아도 관계가 무의미하지는 않다.
3. **인과 오해 방지**: 회귀 = 연관/예측이지, 인과가 아니다. “소득이 높으면 건강 만족도가 높다” ≠ “소득이 건강 만족도를 **일으킨다**”. 역인과, 교란변수, 허위상관 개념 소개.

**집필 방향**
- 회귀 직선의 의미를 **산점도 위에 선 긋기**로 직관 설명 — “이 선이 데이터를 가장 잘 요약하는 직선이다”
- 계수 해석을 **구체적 문장**으로 연습: “교육 연수가 1년 증가하면, 월 소득은 평균 약 50만 원 증가하는 것으로 추정된다.”
- **인과 오해 방지에 수업 시간의 20% 이상 할당** — 이 과목 전체에서 가장 중요한 메시지
- 잔차 분석(residual analysis)은 간략히만: “잔차가 패턴 없이 흩어져야 한다”

**강화 항목**
- ✅ 인과 오해 사례 3가지:
  1. “아이스크림 판매량과 익사 사고 수는 양의 상관” → 교란변수(기온)
  2. “병원에 자주 가는 사람이 더 아프다” → 역인과
  3. “국가별 초콜릿 소비와 노벨상 수 상관” → 허위상관
- ✅ “인과라고 말할 수 있는 조건” 체크리스트: 시간적 선후관계 + 교란변수 통제 + 실험 설계(무작위 배정)

**Copilot 프롬프트 예시**
1. “이 회귀 결과에서 β₁ = 2.3, p = 0.001이다. 이것을 '인과'처럼 읽히지 않게 해석문을 써줘.”
2. “R² = 0.15이면 모델이 쓸모없는 건가? 사회과학에서 R² 기준을 알려줘.”
3. (선택) “이 잔차 플롯을 보고 회귀 가정이 충족되는지 판단해줘.”

**실습 코드 방향**
```python
import statsmodels.api as sm
X = sm.add_constant(df[“교육연수”])
model = sm.OLS(df[“소득”], X).fit()
print(model.summary())
```

### 8주: 1–7주의 핵심을 스스로 설명할 수 있는가? (중간고사)

#### A회차 — 중간고사 대비

**Driving Question**: “1–7주차의 핵심 개념과 분석 흐름을 스스로 설명할 수 있는가?”

**집필 방향**
- 전체 복습이 아니라 **”분석 선택 의사결정”에 집중**: “이 연구 질문에 어떤 분석을 쓸 것인가?”
- 검정 선택 가이드표(5주)를 다시 제시 + 연습 문제 3–4개

### 9주: 여러 변수를 동시에 고려하면 해석이 어떻게 달라지는가?

#### A회차 — 중다회귀(해석/통제의 의미, 한계)

**Driving Question**: “여러 변수를 동시에 고려하면 해석이 어떻게 달라지는가?”

**핵심 개념 (3개)**
1. **중다회귀(multiple regression)**: Y = β₀ + β₁X₁ + β₂X₂ + ... + ε. β₁ = “**다른 변수를 통제한 상태에서** X₁이 1단위 증가할 때 Y의 평균 변화량”. 단순회귀의 β₁과 다를 수 있다.
2. **통제(control)의 의미**: “다른 변수를 일정하게 유지한 상태”라는 표현의 정확한 뜻. 통제 ≠ 인과 해결. 관찰 데이터에서 통제는 “통계적 조정”일 뿐, 실험적 통제가 아니다.
3. **다중공선성(multicollinearity)**: 독립변수끼리 강하게 상관 → 계수 추정이 불안정. VIF(분산팽창인자) > 10이면 주의.

**집필 방향**
- **”단순회귀 → 중다회귀에서 계수가 바뀌는 이유”를 핵심 예제로**: 소득 ~ 교육연수 (단순) vs 소득 ~ 교육연수 + 경력연수 (중다). 교육연수의 계수가 왜 줄어드는가?
- **”통제했다 = 인과를 증명했다”는 오해를 강력히 경고** — 관찰 데이터의 본질적 한계
- VIF 확인을 **필수 절차**로 포함
- 표준화 계수(beta)로 “어떤 변수가 더 영향력이 큰가?” 비교

**강화 항목**
- ✅ 단순회귀 vs 중다회귀 계수 비교 표 (동일 데이터, 계수 변화 시각화)
- ✅ “통제”의 한계: 측정되지 않은 교란변수(omitted variable bias)
- ✅ 변수 선택 가이드: 이론 기반 선택 > 통계적 유의성 기반 선택

**Copilot 프롬프트 예시**
1. “이 회귀 모델의 VIF를 계산하고, 다중공선성 문제가 있는지 판단해줘.”
2. “단순회귀에서 β₁ = 3.5였는데 중다회귀에서 β₁ = 1.2로 줄었다. 왜 이런 일이 생기는지 설명해줘.”
3. (선택) “이 중다회귀 결과를 '통제했다'는 표현 없이 정확하게 해석하는 문장을 써줘.”

**실습 코드 방향**
```python
import statsmodels.api as sm
from statsmodels.stats.outliers_influence import variance_inflation_factor
X = sm.add_constant(df[[“교육연수”, “경력연수”, “성별_더미”]])
model = sm.OLS(df[“소득”], X).fit()
print(model.summary())
# VIF 확인
for i, col in enumerate(X.columns[1:]):
    print(f”{col}: VIF = {variance_inflation_factor(X.values, i+1):.2f}”)
```

**참고문헌 방향**
- Babbie (2021) — 다변량 분석 부분
- statsmodels OLS, VIF 공식 문서
