# 강의계획서(Syllabus): 사회통계(사회조사·통계분석 기초 + 프로젝트, GitHub Copilot 활용)

## 1) 수업개요
사회현상의 **양적 분석**에 필요한 통계적 지식을 습득하기 위한 과목이다. 사회학에서 다루는 제반 통계자료의 처리와 해석을 집중적으로 다루며, 사회학 연구의 통계적 절차를 익혀 **자료 수집–정리–분석–해석–보고**의 전 과정을 경험한다.

## 2) 과목의 핵심(“AI 시대형” 운영 철학)
본 과목은 통계의 기본 논리를 배우는 동시에, GitHub Copilot을 **코딩/디버깅/문장화(해석문 다듬기) 조교**로 활용하여 학습 효율을 높인다. 다만 Copilot은 참고 도구이며, 최종 결론은 **코드 실행 결과와 근거**에 기반해 학생이 판단한다.

## 3) 수업목표 및 학습성과(학기 말 할 수 있어야 하는 것)
- 조사/설문 데이터를 정리하고(결측/코딩/변수 정의), 변수 수준(명목/서열/등간·비율)을 **구분**한다.
- 요약통계, 분할표, 가설검정, 평균차 검정(t/ANOVA), 회귀(단순/중다)를 **적절히 선택·실행**한다.
- 결과를 “질문–가설–분석 선택 이유–결과 근거–결론–한계” 구조로 **짧은 보고서/메모**로 작성한다.
- Copilot을 사용하되, 생성 코드/해석문을 **검증**하고(재현 가능), 부정확한 답을 **반박/수정**할 수 있다.

## 4) 수업운영(주 2회 × 90분)
### 4-1. 회차(90분) 표준 운영
- (0–10분) 오늘의 질문/목표/평가 기준 + 빠른 진단(1문항)
- (10–30분) 미니 강의(핵심 개념 3개 이내 + 예시 1개)
- (30–70분) **조별 실습 과제(수업 내 완결형)**: B회차는 같은 조(2–3인)끼리 과제를 협의·수행하고 조별로 제출
- (70–85분) 정리/오류 패턴 공유 + 해석문 "모범 구조" 제시 + **비판적 읽기**(기존 연구 표/결과의 해석이 적절한지 판단 연습)
- (85–90분) Exit ticket(1문항 + 다음 시간 예고)

### 4-2. 실습 환경(권장)
- 코드/분석: Python(예: pandas, scipy, statsmodels) 또는 R(대체 가능)
- 실행 환경: 로컬 Jupyter 또는 Google Colab(무료)
- AI 도우미: GitHub Copilot(수업 1회차에 설치/사용법 안내)
- 팀 구성: 스튜디오 시간에는 **3명**으로 상호 점검(제출은 기본적으로 개인)

### 4-3. 1회차(첫 수업) 배치 자료
- 첫 수업은 `깃허브.md`를 사용하여 **Copilot 설치/로그인/기본 사용법**을 안내하고, 이후 수업에서 동일한 규칙으로 활용한다.

## 5) 수강 전제(권장)
- 원칙적으로 **사회조사방법론 1** 또는 이에 준하는 과목을 수강한 학생
- 표/그래프 읽기, 파일 관리 등 기본 컴퓨터 활용 능력

## 6) 평가방법
| 평가항목 | 비율 |
|---|---:|
| 매주 과제(수업 내 과제/프로젝트 체크포인트) | 40% |
| 중간고사 | 40% |
| 수업참여도 | 20% |

### 6-1. 중간고사 형식(객관식+주관식 혼합)
- 시간: 90분
- 구성: 객관식 60% + 주관식 40%
  - 객관식: 개념/가정/용어, 척도 구분, 어떤 분석을 선택해야 하는지
  - 주관식: 짧은 결과표/그래프/코드 조각을 보고 ① 분석 선택 이유 ② 핵심 수치/패턴 근거 ③ 결론 ④ “의미하지 않는 것(인과 아님 등)” 포함 해석문(5–7문장)
- 시험 중 AI 사용: **Copilot/인터넷 사용 금지**(분쟁 예방을 위해 명시)
- 출제 범위: 1–8주차 전체

### 6-2. 매주 과제(평가) 운영 및 반환
- 매주 1회, 수업시간 내 완결을 원칙으로 한다.
- 1–10주: 개인 과제(기법 습득 + 미니 적용)
- 11–14주: **3인 팀프로젝트 체크포인트(팀 제출)** + 팀원별 기여 3줄
- 제출물(표준 형식):
  - (A) 분석 질문(1–2문장)
  - (B) 변수/척도(명목·서열·등간/비율)
  - (C) 근거(핵심 결과표/그래프/수치 1–2개 또는 3개 이내)
  - (D) 해석문(5–7문장, 비인과 표현 포함)
  - (E) Copilot 사용 요약(2–3줄: 무엇을 요청했고 무엇을 검증했는가)
- 반환(피드백) 원칙: **다음 주 첫 수업 시작 전/또는 첫 10분 내** 점수+루브릭+핵심 코멘트 1개로 반환한다.

### 6-3. 팀프로젝트 평가 세부 기준(11–14주)
- 팀 과제(과제 10–13)는 **팀 공통 점수(70%) + 개인 기여 점수(30%)**로 구성한다.
- 팀 공통 점수: 분석의 타당성, 근거 제시, 해석의 정확성, 재현 가능성(루브릭 4항목 기준)
- 개인 기여 점수: 매 체크포인트 제출 시 “팀원별 기여 3줄”을 근거로 평가한다.
  - 기여 내용이 불균형한 경우(예: 한 명의 기여가 현저히 부족), 교수자가 개별 면담 후 개인 점수를 **차등 조정**한다.
  - 무임승차가 확인되면 해당 학생의 팀 과제 점수를 **최대 50%까지 감점**할 수 있다.

### 6-4. 수업참여도(20%)
- 수업참여도는 출석 기반으로 평가한다.
- **결석**: 1회당 **2점 차감**
- **지각**(수업 시작 후 15분 이내 입실): 1회당 **1점 차감**
- 공결(학교 공식 행사, 질병 등 증빙 가능한 경우)은 차감 대상에 포함하지 않는다.

### 6-5. 결석 시 과제 처리
- 수업 내 완결형 과제를 전제로 하므로, **결석 시 해당 주 과제는 원칙적으로 0점** 처리한다.
- 단, 학기 중 **1회에 한해 대체 과제**를 허용한다(사유 불문, 다음 주 수업 전까지 제출).
- 공결은 대체 과제 횟수에 포함하지 않는다.

## 7) Copilot 사용 정책(윤리·검증·프라이버시)
- 허용: 개념/절차 설명 요청, 디버깅 절차, 코드 초안, 문장 다듬기, 테스트/검증 코드 작성 보조
- 금지: 개인정보/민감정보 입력, 시험 답안 생성/복사 제출, 데이터 위조, 타인 과제 복제
- 원칙: Copilot 출력은 참고이며, 최종 결론은 **실행 결과와 근거**로 학생이 판단한다.

## 8) 팀프로젝트(11–14주, 3인 팀)
### 8-1. 팀프로젝트 목표
**10주차까지 핵심 분석 흐름(데이터 준비→요약→관계/검정→회귀→해석/보고)을 완료**하고, 11–14주에는 3인 1팀으로 하나의 사회과학적 질문을 정해 **분석–해석–보고**를 완결한다.

### 8-2. 팀 구성 및 운영 원칙
- 팀 구성: **3명**
- 팀프로젝트 기간(11–14주)에는 수업의 B회차(스튜디오)가 팀 작업 시간이며, 주간 과제는 **팀 체크포인트**로 제출한다.
- 무임승차 방지: 매 체크포인트 제출 시, 팀원별 기여(본인 작업 3줄)를 함께 제출한다(형식은 수업에서 안내).

### 8-3. 최종 산출물(권장)
- 재현 가능한 노트북/스크립트 + 실행 방법(README) 1세트
- 2쪽 내외 요약 메모 1개(질문, 자료, 방법, 결과, 한계)
- 5분 내외 팀 발표 자료(또는 1장 포스터)

## 10) 주차/회차별 강의계획(10주 완주 + 4주 팀프로젝트)
> 10주차까지 핵심 개념과 분석 흐름을 “완주”하고, 11–14주는 3인 팀프로젝트 스튜디오로 운영한다.

| 주차 | A회차(개념/도입) | B회차(스튜디오/과제: 수업 내 완결형) |
|---:|---|---|
| 1주 | 오리엔테이션: AI 도구를 통계 학습의 조교로 어떻게 활용할까?(`깃허브.md`) | 환경 점검 + 미니 실습(개인 과제 1) |
| 2주 | 이 데이터를 믿을 수 있는가? — 데이터 리터러시 + 코딩/결측/변수 정의 | 데이터 정리 체크포인트(개인 과제 2) |
| 3주 | 데이터의 전체 모습을 어떻게 파악하고 표현할까? — 요약·분포·산포 + 해석문 구조 | EDA 체크포인트(개인 과제 3) |
| 4주 | 두 범주형 변수가 관련 있는가? — 분할표·카이자승 | 범주형 분석 체크포인트(개인 과제 4) |
| 5주 | 표본의 패턴이 모집단에도 존재하는가? — 기술→추론 전환 + 가설검정 프레임 | “가설–분석 선택–결론” 문장화(개인 과제 5) |
| 6주 | 집단 간 평균이 '진짜' 다른가? — t-검정/ANOVA | 평균차 분석 체크포인트(개인 과제 6) |
| 7주 | 한 변수가 다른 변수를 얼마나 예측하는가? — 단순회귀(인과 오해 방지) | 단순회귀 체크포인트(개인 과제 7) |
| 8주 | 여러 변수를 동시에 고려하면 해석이 어떻게 달라지는가? — 중다회귀 | 중다회귀 체크포인트 + **팀프로젝트 데이터 후보 조사**(개인 과제 8) |
| 9주 | 중간고사 대비(핵심 개념/해석 연습) | **중간고사(객관식+주관식)** |
| 10주 | 질문부터 해석까지 혼자 완주할 수 있는가? — 미니 완주 프로젝트 | 개인 미니 종합(요약 메모 v1)(개인 과제 9) |
| 11주 | 팀으로 하나의 질문에 답하려면 무엇을 먼저 정해야 하는가? — 킥오프(8주 후보 기반) | 팀 체크포인트 1: 질문/가설/분석계획(팀 과제 10) |
| 12주 | 분석 결과를 다른 사람이 재현할 수 있는가? — 분석 실행/검증 | 팀 체크포인트 2: 핵심 결과 2개 + 해석 초안(팀 과제 11) |
| 13주 | 분석 결과를 설득력 있게 전달하려면? — 보고서/발표 구성 | 팀 체크포인트 3: 최종 산출물 리허설(팀 과제 12) |
| 14주 | 쇼케이스 준비(피드백 반영) | 팀프로젝트 쇼케이스 + 최종 제출(팀 과제 13) |

> 기말고사는 14주 이후 학사 시험기간 중 실시한다(정확한 일시는 학사 일정 확정 후 공지).
