## 1. 빠른 진단 정답

**질문**: "Copilot에게 코드를 요청할 때, 구체적인 프롬프트와 모호한 프롬프트 중 어느 쪽이 더 좋은 결과를 줄까? 이유는?"

**정답**: 구체적인 프롬프트가 더 좋은 결과를 준다. 목표/제약/출력 형식을 명시할수록 Copilot이 의도에 맞는 코드를 생성한다.

**해설**: 모호한 프롬프트(예: "CSV를 불러와")는 Copilot이 맥락을 추측해야 하므로, 사용자가 원하지 않는 방식으로 코드를 생성할 수 있다. 구체적인 프롬프트(예: "pandas로 CSV를 불러와 결측치를 확인하고, 변수별 요약통계를 출력하는 코드를 짜줘")는 라이브러리, 분석 내용, 출력 형식을 명시하므로 의도에 부합하는 코드가 나올 확률이 높다.

## 3. 과제 2 기대 결과

### 결측치 현황

| 변수명 | 결측치 수 |
|---|---:|
| 성별 | 0 |
| 연령 | 0 |
| 교육수준 | 0 |
| 소득 | 8 |
| 생활만족도 | 3 |
| 정치관심도 | 0 |

### 척도 분류 정답

| 변수명 | 올바른 척도 | 해설 |
|---|---|---|
| 성별 | 명목(nominal) | 범주 간 순서 없음 |
| 연령 | 비율(ratio) | 절대 영점(0세) 존재, 연속 수치 |
| 교육수준 | 서열(ordinal) | 고졸이하 < 대졸 < 대학원이상 순서 있으나 간격이 동일하지 않음 |
| 소득 | 비율(ratio) | 절대 영점(0원) 존재, 연속 수치 |
| 생활만족도 | 서열(ordinal) | 1--5점 순서 있으나, "1점과 2점의 차이"와 "4점과 5점의 차이"가 동일하다고 보장할 수 없음 |
| 정치관심도 | 서열(ordinal) | 생활만족도와 동일한 논리 |

> **Copilot 오답 가능성**: Copilot이 생활만족도/정치관심도를 "등간(interval)"이라고 답하는 경우가 잦다. 리커트 5점 척도를 등간으로 보는 것은 학계에서도 논쟁이 있으나, 보수적으로는 서열로 분류하는 것이 적절하다. 학생이 Copilot과 다른 판단을 하고 근거를 제시했다면 높게 평가한다.

### 기대 코드 (바이브코딩 결과 예시)

```python
# 변수별 결측치 확인
print(df.isnull().sum())
```

### (E) Copilot 사용 요약 (모범)

## 5. 흔한 오류 3개 + 확인법

### 오류 1: 파일 경로 오류

`pd.read_csv("social_survey_w01.csv")`를 실행했을 때 `FileNotFoundError`가 나오는 경우이다. CSV 파일이 Python 파일과 **같은 폴더**에 있는지 확인한다. 다른 폴더에 있다면 전체 경로를 지정하거나, 파일을 같은 폴더로 옮긴다.

**확인법**: Copilot에게 "FileNotFoundError가 발생했다. 현재 작업 디렉토리를 확인하는 코드를 짜줘"라고 요청하면, `import os; print(os.getcwd())` 코드를 얻을 수 있다.

```python
# 경로를 직접 지정하는 예
df = pd.read_csv("/Users/학생이름/Documents/social_survey_w01.csv")
```

### 오류 2: 한글 인코딩 깨짐

CSV 파일을 불러왔는데 한글이 깨져 보이는 경우이다. `encoding` 옵션을 지정하여 해결한다.

**확인법**: 데이터를 출력(`print(df.head())`)했을 때 한글 컬럼명이나 값이 `????` 또는 깨진 문자로 나오는지 확인한다.

```python
# UTF-8으로 시도
df = pd.read_csv("social_survey_w01.csv", encoding='utf-8')
# 안 되면 cp949로 시도
df = pd.read_csv("social_survey_w01.csv", encoding='cp949')
```

### 오류 3: `df.describe()`가 숫자 변수만 보여줌

`df.describe()`는 기본적으로 숫자형 변수만 출력한다. 범주형 변수(성별, 교육수준 등)의 요약도 보려면 `include='all'` 옵션을 사용한다.

**확인법**: `describe()` 출력에 성별/교육수준이 없다면 이 오류이다.

```python
print(df.describe(include='all'))
```

## 6. 비판적 읽기 --- 토론 가이드 + 모범 답안

### 가상 Copilot 답변 (B.md에 제시된 것)

> **학생의 질문**: "이 데이터에 대해 알려줘."
>
> **Copilot의 답변**: "이 데이터의 모든 변수는 정규분포를 따릅니다. 결측치는 무시해도 됩니다. 이 데이터로 인과관계를 분석할 수 있습니다."

### 토론 가이드

아래 3가지 문제점을 확인한다.

### 모범 답안 (문제점 3가지)

**문제점 1: "모든 변수는 정규분포를 따른다"**

명목 변수(성별)와 서열 변수(교육수준)는 정규분포 개념이 적용되지 않는다. 연속 변수(연령, 소득)도 실제로 정규분포를 따르는지는 히스토그램이나 검정으로 확인해야 한다. Copilot의 답변을 그대로 믿으면 안 된다.

**문제점 2: "결측치는 무시해도 된다"**

결측치의 양과 패턴에 따라 처리 방법이 달라진다. 무조건 무시하면 분석 결과가 편향될 수 있다. 소득 변수의 결측 8건(4%)은 적지 않은 비율이며, 결측이 "무작위"인지 "특정 패턴(예: 고소득자가 응답을 거부)"인지에 따라 처리 방법이 다르다. 다음 주(2주차)에서 이 주제를 본격적으로 다룬다.

**문제점 3: "인과관계를 분석할 수 있다"**

설문 데이터(관찰 데이터)로는 인과관계를 직접 확인할 수 없다. 변수 간 연관(association)이나 관계(relationship)는 탐색할 수 있지만, "A가 B를 일으킨다"고 결론 내리려면 실험 설계나 교란변수 통제가 필요하다.

### 핵심 교훈

이처럼 Copilot은 자신 있게 **틀린 답변**을 할 수 있다. 이것이 A회차에서 배운 **검증 원칙**이 중요한 이유이다. Copilot의 답변을 항상 의심하고, 코드 실행 결과로 확인하는 습관을 들여야 한다.

## 8. 해석문 모범 구조 (채점 참고)

### 해석문(D항) 채점 포인트

| 항목 | 배점 기준 |
|---|---|
| (분석목적) 명시 | 무엇을 확인하기 위한 분석인지 1문장으로 서술 |
| (핵심근거) 수치 제시 | 데이터 크기, 평균/표준편차, 결측치 수를 구체적으로 기록 |
| (결론) 판단 | 결측치 처리 필요성 등 다음 단계를 언급 |
| (주의) 한계 | 가상 데이터의 한계, 일반화 불가 등 명시 |

### 감점 사유

- 수치가 실행 결과와 일치하지 않음 (임의로 기재한 경우)
- 해석문에 인과 표현 사용 ("소득이 만족도에 영향을 미친다" 등)
- Copilot 사용 요약(E항)이 없거나, 사용 과정이 전혀 기록되지 않은 경우
