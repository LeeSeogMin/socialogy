# W12. 분석 결과를 다른 사람이 재현할 수 있는가? — 분석 실행과 재현성

> 팀의 분석을 실행하고, 제3자가 동일한 결과를 재현할 수 있도록 기록할 수 있는가?

**학습 목표**
1. **재현성(reproducibility)**의 의미를 이해하고, 분석의 재현성을 확보하기 위한 체크리스트를 적용할 수 있다.
2. 팀 분석을 실행하여 **핵심 결과 2개**를 도출하고, A-E 형식으로 해석 초안을 작성할 수 있다.
3. 코드 리뷰를 통해 **다른 팀원이 동일한 결과를 얻을 수 있는지** 확인할 수 있다.

---

## A회차: 재현성과 코드 리뷰

### 재현성이란

**재현성(reproducibility)**이란 같은 데이터와 같은 코드를 사용했을 때, **누가 실행해도 동일한 결과**가 나오는 것을 말한다. 사회과학 연구에서 재현성은 분석 결과의 신뢰성을 뒷받침하는 핵심 조건이다.

재현성이 확보되지 않으면:
- 같은 팀원이 실행해도 다른 결과가 나올 수 있다.
- 교수나 동료가 결과를 검증할 수 없다.
- "이 결과를 믿을 수 있는가?"라는 질문에 답할 수 없다.

---

### 재현성 체크리스트

| 항목 | 확인 내용 | 확인 |
|---:|---|---|
| 1 | **데이터 파일 명시**: 사용한 데이터 파일명과 경로가 명확한가? | [ ] |
| 2 | **실행 순서 명확화**: 코드를 위에서 아래로 순서대로 실행하면 결과가 나오는가? | [ ] |
| 3 | **랜덤 시드 설정**: 무작위 요소가 있다면 `random_state` 또는 `np.random.seed()`를 설정했는가? | [ ] |
| 4 | **라이브러리 버전 기록**: 사용한 패키지와 버전이 기록되어 있는가? | [ ] |
| 5 | **중간 결과 확인**: 주요 단계마다 출력을 확인할 수 있는가? | [ ] |

---

### 코드 리뷰 절차

팀 내에서 아래 절차로 코드 리뷰를 수행한다.

1. **작성자가 아닌 팀원**이 노트북을 처음부터 끝까지 실행한다.
2. 실행 중 오류가 발생하면 그 위치와 내용을 기록한다.
3. 최종 결과가 작성자의 결과와 **동일한지** 비교한다.
4. 동일하지 않은 경우, 원인을 찾아 수정한다.

쉽게 말해서, "내가 아닌 다른 사람이 실행해도 같은 결과가 나오는가?"를 직접 확인하는 과정이다.

---

### 과제 구성

#### ① 분석 실행 (20분)

11주에서 세운 분석 계획에 따라 검정을 실행한다.

- 기술통계를 확인한다.
- 검정의 가정을 확인한다.
- 검정을 실행하고 **핵심 결과 2개**(표/그래프/수치)를 정리한다.

#### ② 해석 초안 작성 (10분)

핵심 결과 2개 각각에 대해 A-E 형식으로 해석 초안을 작성한다.

- (A) 분석 질문
- (B) 사용 변수/척도
- (C) 결과 근거 (표/그래프 또는 수치)
- (D) 해석문 5-7문장 (4단계: 분석목적 → 핵심근거 → 결론 → 주의/한계)
- (E) Copilot 사용 요약

#### ③ 재현성 확인 및 기여 기록 (10분)

- 작성자가 아닌 팀원이 노트북을 실행하여 동일 결과를 확인한다.
- 재현성 체크리스트 5개 항목을 점검한다.
- 팀원별 기여 3줄을 작성한다.

---

### 흔한 오류 3가지

| 오류 | 확인법 |
|---|---|
| 데이터 파일 경로가 본인 컴퓨터에만 맞게 되어 있다 | 상대 경로를 사용하거나, 데이터 파일을 노트북과 같은 폴더에 둔다 |
| 코드 셀 실행 순서가 뒤섞여 있다 | "Restart & Run All"로 전체 실행이 되는지 확인한다 |
| 가정 확인 없이 검정 결과만 보고한다 | 기대빈도 ≥ 5, 정규성, 등분산 등 해당 검정의 가정을 먼저 확인한다 |
