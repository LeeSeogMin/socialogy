# W13. 분석 결과를 설득력 있게 전달하려면? — 보고서 구성과 발표 리허설

> 분석 결과를 사회과학 보고서 구조에 맞추어 전달하고, 동료 피드백을 반영할 수 있는가?

**학습 목표**
1. 사회과학 보고서 구조(질문 → 데이터/방법 → 결과 → 해석 → 한계)에 맞추어 **2쪽 요약 메모 초안**을 작성할 수 있다.
2. 5분 발표를 통해 분석의 **핵심 메시지("So what?")**를 전달할 수 있다.
3. 짝 팀 피드백을 통해 보고서와 발표의 **개선점을 식별**할 수 있다.

---

## A회차: 보고서와 발표 구성

### 사회과학 보고서 구조

최종 제출물인 2쪽 요약 메모는 아래 구조를 따른다.

| 섹션 | 분량 | 핵심 내용 |
|---|---|---|
| **1. 연구 질문** | 2-3문장 | 무엇을 알고 싶은가? 왜 중요한가? |
| **2. 데이터와 방법** | 4-5문장 | 어떤 데이터를 사용했는가? 변수와 척도는? 어떤 기법을 선택했으며 왜? |
| **3. 결과** | 5-7문장 + 표/그래프 | 핵심 수치는? 가설 검정 결과는? |
| **4. 해석과 논의** | 3-4문장 | 이 결과가 무엇을 의미하는가? "So what?" |
| **5. 한계** | 2-3문장 | 이 분석이 말할 수 없는 것은? 데이터의 제약은? |

---

### "So what?" — 핵심 메시지

분석 결과를 나열하는 것만으로는 충분하지 않다. 듣는 사람이 가장 궁금해하는 것은 **"그래서 무엇을 알 수 있는가?"**이다.

핵심 메시지를 만드는 3단계:
1. **결과 요약**: "분석 결과, ___와 ___ 사이에 통계적으로 유의한 관련/차이가 있었다(또는 없었다)."
2. **의미 부여**: "이는 ___를 시사한다."
3. **한계 명시**: "다만, 이 결과가 ___를 의미하지는 않는다."

---

### 5분 발표 구성 가이드

| 시간 | 내용 |
|---:|---|
| 0-1분 | 연구 질문 + 왜 중요한가 |
| 1-2분 | 데이터·방법 요약 |
| 2-4분 | 핵심 결과 (표/그래프 1-2개로 설명) |
| 4-5분 | "So what?" + 한계 |

**발표 시 주의사항**:
- 슬라이드에 숫자를 나열하지 말고, **핵심 그래프/표 1-2개**로 설명한다.
- 인과 표현을 사용하지 않는다. "A가 B를 초래한다" (X) → "A와 B 사이에 유의한 관련이 있다" (O)
- "모르겠다", "확신할 수 없다"는 솔직한 표현이 학술적으로 올바르다.

---

### 짝 팀 피드백 가이드

다른 팀의 발표를 듣고 아래 3가지 질문에 답한다.

1. 연구 질문이 **명확하고 구체적**인가?
2. 결과 근거(표/그래프/수치)가 결론을 **충분히 뒷받침**하는가?
3. 한계를 **솔직하게** 언급했는가?

피드백은 "잘한 점 1개 + 개선 제안 1개" 형식으로 작성한다.

## B회차: 팀 체크포인트 3 — 과제 12

### 과제 안내

| 항목 | 내용 |
|---|---|
| 과제 유형 | **팀 과제** (3인, 조별 1부 제출) |
| 소요 시간 | 40분 |
| 제출 형식 | 2쪽 요약 메모 초안 + 발표 리허설 기록 + 피드백 |
| 제출처 | Google Classroom |

---

### 과제 구성

#### ① 2쪽 요약 메모 초안 작성 (15분)

위 보고서 구조(5개 섹션)에 맞추어 2쪽 요약 메모 초안을 작성한다.

- 12주 체크포인트에서 제출한 핵심 결과와 해석 초안을 기반으로 한다.
- 표/그래프는 최종 버전으로 정리한다.

#### ② 5분 발표 리허설 (15분)

- 팀 내에서 발표를 연습한다.
- 발표 시간이 5분을 초과하지 않는지 확인한다.
- 각 팀원이 발표의 일부를 담당한다.

#### ③ 교차 피드백 및 마무리 (10분)

- 짝 팀과 발표를 교환하고, 피드백을 주고받는다.
- 재현성 최종 체크: 노트북 "Restart & Run All" 확인
- 팀원별 기여 3줄을 작성한다.

---

### Copilot 활용 안내

**추천 프롬프트**:
- "아래 분석 결과를 바탕으로 2쪽 요약 메모의 '결과' 섹션 초안을 작성해줘. 해석문 4단계 구조를 따라줘."
- "이 발표 스크립트에서 인과 표현이 있는지 확인하고, 수정 제안을 해줘."

주의: Copilot의 초안은 **참고용**이다. 최종 문장은 팀이 직접 작성하고 검증한다.
