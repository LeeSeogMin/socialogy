## 1. 빠른 진단 정답

**질문**: "결측치가 있는 행을 무조건 삭제하면 어떤 문제가 생길 수 있는가?"

**정답**: 결측이 특정 집단에 편중되어 있으면(**MAR**(Missing At Random) 또는 **MNAR**(Missing Not At Random)), 삭제 시 표본이 편향되어 분석 결과가 왜곡될 수 있다. 또한 결측 비율이 높으면 데이터가 크게 줄어들어 통계적 **검정력**(statistical power)이 약해진다.

**보충 설명**: A회차에서 배운 결측 유형(MCAR/MAR/MNAR)을 복기한다. 결측이 무작위(MCAR)가 아닌 경우, 삭제는 체계적 편향을 초래한다.

## 2. 과제 1 기대 결과: 결측치·이상치 탐색

### 기대 코드 예시

```python
import pandas as pd

df = pd.read_csv("social_survey_w02_raw.csv")

# 결측치 현황
print("=== 결측치 현황 ===")
print(df.isnull().sum())
print(f"\n전체 결측 비율: {df.isnull().mean().mean()*100:.1f}%")

# 이상치 확인 (수치형 변수)
print("\n=== 수치형 변수 기술통계 ===")
print(df.describe())

# 연령 이상치 필터링
print("\n=== 연령 이상치 ===")
print(df[(df["연령"] > 100) | (df["연령"] < 0)])

# 소득 이상치 필터링 (극단값)
print("\n=== 소득 기술통계 ===")
print(df["소득"].describe())
```

### 기대 결과 수치

**현황 보고표 (기대 정답)**:

| 변수명 | 척도 | 결측 수 | 결측 비율 | 이상치 여부 | 이상치 값 |
|---|---|---|---|---|---|
| 성별 | 명목 | 5 | 2.5% | X | |
| 연령 | 비율 | 0 | 0% | O | 999, -1, 200 |
| 교육수준 | 서열 | 8 | 4.0% | X | |
| 소득 | 비율 | 25 | 12.5% | O | 극단값 2건 |
| 생활만족도 | 서열 | 10 | 5.0% | X | |
| 정치관심도 | 서열 | 0 | 0% | X | |

**핵심 확인 포인트**:
- 연령 이상치 3건(999, -1, 200)은 `describe()`의 min/max에서 바로 발견 가능하다.
- 소득 이상치 2건은 극단적으로 높은 값으로, 분포의 꼬리에 위치한다.
- 소득의 결측 비율(12.5%)이 가장 높다. 이 수준이면 단순 삭제보다 대체를 고려해야 한다.

## 3. 과제 2 기대 결과: 결측치·이상치 처리 + 전후 비교

### 기대 코드 예시

```python
import pandas as pd

df = pd.read_csv("social_survey_w02_raw.csv")

# 원본 보관
df_raw = df.copy()

# 처리 전 기술통계 저장
before = df.describe()

# 이상치 처리: 범위 밖 값을 NaN으로 변환
df.loc[df["연령"] > 100, "연령"] = pd.NA
df.loc[df["연령"] < 0, "연령"] = pd.NA
# 소득 이상치도 처리 (극단값을 NaN으로)
# 구체적 기준은 데이터에 따라 다름 — IQR 방법 또는 절대 기준 사용

# 결측치 처리: 수치형은 중앙값, 범주형은 최빈값
df["연령"] = df["연령"].fillna(df["연령"].median())
df["소득"] = df["소득"].fillna(df["소득"].median())
df["생활만족도"] = df["생활만족도"].fillna(df["생활만족도"].median())
df["교육수준"] = df["교육수준"].fillna(df["교육수준"].mode()[0])
df["성별"] = df["성별"].fillna(df["성별"].mode()[0])

# 처리 후 기술통계
after = df.describe()
print("=== 처리 전 ===")
print(before)
print("\n=== 처리 후 ===")
print(after)
print(f"\n남은 결측: {df.isnull().sum().sum()}건")
```

### 기대 결과 해설

**전후 비교표 (기대 방향)**:

| 항목 | 처리 전 | 처리 후 | 변화 방향 |
|---|---|---|---|
| 연령 평균 | 높음 (999 등 포함) | 크게 하락 | 이상치 제거 효과 |
| 연령 표준편차 | 매우 큼 | 크게 감소 | 이상치 제거 효과 |
| 소득 평균 | 이상치 포함 | 약간 하락 | 극단값 제거 효과 |
| 소득 표준편차 | 큼 | 감소 | 이상치 제거 + 중앙값 대체 |
| 전체 결측 수 | 48건 이상 | 0건 | 전부 대체 완료 |
| 유효 데이터 수 | (결측 제외) | 200 (전체) | 행 삭제 없이 대체 |

**핵심 해설**:
- 연령의 이상치(999, -1, 200)를 NaN으로 바꾼 뒤 중앙값으로 대체하면, 평균이 극적으로 하락한다. 이것이 이상치가 평균에 미치는 영향을 보여주는 핵심 포인트이다.
- 중앙값 대체 후 표준편차가 감소하는 이유는, 결측값을 모두 "중심값"으로 채우면 데이터가 중앙으로 몰리기 때문이다.
- 소득 결측 25건(12.5%)을 중앙값으로 대체하면 소득 분포가 실제보다 덜 퍼진 형태가 된다. 이 한계를 학생이 인지하는 것이 중요하다.

## 4. 모범 보고서 (A–E)

### (A) 분석 질문

"이 데이터(social_survey_w02_raw.csv)에서 어떤 결측치와 이상치가 존재하며, 이를 처리한 후 기술통계가 어떻게 변하는가?"

### (B) 사용 변수/척도 + 결측/이상치 현황

| 변수명 | 척도 | 결측 | 이상치 |
|---|---|---|---|
| 성별 | 명목 | 5건(2.5%) | 없음 |
| 연령 | 비율 | 0건 | 3건(999, -1, 200) |
| 교육수준 | 서열 | 8건(4.0%) | 없음 |
| 소득 | 비율 | 25건(12.5%) | 2건(극단값) |
| 생활만족도 | 서열 | 10건(5.0%) | 없음 |
| 정치관심도 | 서열 | 0건 | 없음 |

### (C) 결과 근거

Copilot 생성 코드(과제 1, 과제 2에서 실행한 코드 전문) + 처리 전후 기술통계 비교표(과제 2 전후 비교표)를 포함한다.

### (D) 해석문 (모범 답안)

> **(분석목적)** 원시 데이터의 결측치·이상치를 확인하고, 분석 가능한 형태로 정리하기 위해 탐색적 점검을 수행했다.
>
> **(핵심근거)** 6개 변수 중 4개에서 결측이 발견되었으며, '소득' 변수의 결측 비율이 12.5%로 가장 높았다. '연령' 변수에서 범위 밖 값(999, -1, 200)이 3건 발견되어 NaN으로 처리했다. '소득' 변수에서도 극단적 이상치 2건이 발견되었다.
>
> **(결론)** 이상치 제거 후 연령의 평균이 크게 하락하여, 이상치가 평균을 심하게 왜곡하고 있었음을 확인했다. 결측치를 수치형은 중앙값, 범주형은 최빈값으로 대체한 후 전체 결측은 0건이 되었으며, 소득의 표준편차가 감소했다.
>
> **(주의)** 중앙값 대체는 분산을 줄이는 효과가 있으므로, 실제 분석 시에는 결측 유형에 따른 적절한 처리 방법을 선택해야 한다. 특히 '소득' 변수의 결측이 MNAR(고소득자나 저소득자가 응답을 회피)일 가능성을 고려해야 한다. 중앙값 대체는 간편하지만, 결측이 무작위가 아니면 체계적 편향을 남길 수 있다.

### (E) Copilot 사용 요약 (모범 답안)

> Copilot에게 "결측치를 중앙값으로 대체하면 분포에 어떤 영향이 있는지 설명해줘"라고 요청했다. Copilot은 "중앙값 대체는 분산을 줄인다"고 답했다. 채택한다. 실제로 처리 전후 기술통계를 비교한 결과 소득의 표준편차가 감소하여 Copilot의 설명이 실행 결과와 일치했기 때문이다.

## 5. 흔한 오류 3개 + 확인법

### 오류 1: 이상치를 발견하지 못함

`df.describe()`의 min/max 값을 반드시 확인한다. 연령이 999나 -1인 경우는 명백한 이상치이다. 의심되면 직접 필터링하여 확인한다.

```python
# 연령이 100 이상이거나 0 미만인 행을 찾는다
print(df[(df["연령"] > 100) | (df["연령"] < 0)])
```

**주의**: `describe()`를 실행했다면 min=-1, max=999를 그냥 넘기지 않아야 한다. min과 max를 먼저 확인하는 습관이 중요하다.

### 오류 2: 원본 데이터를 덮어쓰기

처리 과정에서 원본 데이터를 직접 수정하면, 나중에 처리 전후를 비교할 수 없다. 원본은 별도로 보관하고, 처리본은 새 변수나 새 파일로 저장하는 것이 좋다.

```python
# 원본 보관
df_raw = df.copy()
# 처리는 df에 수행
df.loc[df["연령"] > 100, "연령"] = pd.NA
```

**주의**: `before = df.describe()`를 처리 전에 실행하지 않으면 비교 자체가 불가능하다. 처리 전 기술통계를 반드시 먼저 저장한다.

### 오류 3: 처리 전후 비교를 생략

결측치나 이상치를 처리한 뒤 "남은 결측 0건"만 확인하고 끝내는 경우가 있다. 반드시 **평균, 표준편차, 분포가 어떻게 변했는지** 비교해야 한다. 처리가 데이터의 특성을 왜곡하지는 않았는지 점검하는 것이 핵심이다.

**주의**: 결측을 채운 뒤 평균과 표준편차가 어떻게 변했는지 반드시 확인한다. 전후 비교가 없으면 처리의 영향을 알 수 없다.

## 6. 비판적 읽기 자료 + 토론 가이드

### 사례

> *"한 연구에서 소득 변수의 결측치(15%)를 모두 삭제하고 평균 소득을 분석했다. 분석 결과, 이 지역 주민의 평균 소득은 350만 원으로 나타났다."*

### 토론 질문

"이 처리 방법에 어떤 문제가 있을 수 있는가?" — 조별로 2–3분 토론 후 전체 공유한다.

### 문제점

1. **MNAR 가능성**: 고소득자가 소득 공개를 꺼려 미응답했다면, 고소득 데이터가 삭제되어 평균 소득이 실제보다 낮게 추정된다. 반대로 저소득자가 미응답했다면 평균이 높게 추정된다. 결측의 원인을 모르면 삭제의 영향을 판단할 수 없다.

2. **표본 크기 감소**: 15%를 삭제하면 유효 데이터가 줄어들어 추후 통계 검정의 **검정력**(statistical power)이 약해진다. 표본이 작을수록 우연에 의한 변동이 커져서 결과의 신뢰도가 낮아진다.

3. **대안 미검토**: 삭제 외에도 중앙값 대체, **다중 대체**(multiple imputation) 등의 방법이 있다. 이러한 대안을 검토하지 않고 바로 삭제를 선택한 것은 정당화가 필요하다.
