# W01A. AI 도구를 통계 학습의 조교로 어떻게 활용할 수 있을까?

> AI 도구(Copilot)를 "정답 생성기"가 아니라 "코딩·디버깅·해석문 다듬기 조교"로 활용하려면 어떤 원칙이 필요할까?

**학습 목표**
1. GitHub 계정을 만들고 VS Code에 **GitHub Copilot**을 설치·활성화하여 기본 사용 준비를 완료할 수 있다.
2. Copilot의 역할을 "정답 생성기"가 아닌 **"코딩 조교"**로 정의하고, 검증 원칙과 프라이버시 규칙을 설명할 수 있다.
3. Copilot Chat에 질문 템플릿을 사용하여 코드를 요청하고, 실행 결과로 **검증**하는 과정을 수행할 수 있다.

---

## 오늘의 질문/목표/진단

### 오늘의 질문

이번 학기에 우리는 사회통계를 배운다. 통계 수업에서는 코드를 짜고, 에러를 고치고, 결과를 해석하는 과정이 반복된다. 이 과정에서 시간이 가장 많이 걸리는 부분은 어디일까? 많은 경우 "코드 작성"과 "에러 수정"에 시간을 빼앗기고, 정작 중요한 "결과 해석"과 "판단"에는 시간이 부족해진다.

**GitHub Copilot**은 이 문제를 완화해주는 도구이다. Copilot이 코드 작성과 디버깅을 도와주면, 학생은 더 많은 시간을 **통계적 사고와 해석**에 집중할 수 있다. 다만, Copilot은 "정답 생성기"가 아니다. Copilot이 만든 코드와 설명은 반드시 **실행 결과로 검증**해야 하며, 최종 판단은 학생 본인이 내린다.

오늘은 첫 수업이므로 통계 내용을 다루지 않는다. 환경 설정(Copilot 설치)과 수업 규칙 공유에 집중한다.

## 핵심 개념 학습

### 왜 통계 수업에서 AI를 쓰는가

통계 실습의 흐름은 대체로 "코드 작성 → 실행 → 에러 수정 → 결과 확인 → 해석 작성"이다. 이 중 코드 작성과 에러 수정에 드는 시간이 전체의 절반 이상을 차지하는 경우가 많다. Copilot은 이 앞단의 시간을 줄여줌으로써, 학생이 **결과 해석과 판단**에 더 많은 시간을 쓸 수 있게 한다.

그렇다고 코딩을 전혀 몰라도 된다는 뜻은 아니다. Copilot의 제안이 맞는지 틀린지를 판단하려면 코드를 읽고 이해하는 능력이 필요하다. Copilot은 **조교**이지, 학생을 대신하는 것이 아니다.

---

### 핵심 개념 1: Copilot의 역할 정의 — "코딩 조교"

**GitHub Copilot**은 코드 편집기(VS Code 등)에서 작동하는 AI 코딩 도우미이다. 크게 두 가지 기능을 제공한다.

1. **코드 완성(Autocomplete)**: 코드를 작성하다가 멈추면, 다음에 올 코드를 회색 글씨(ghost text)로 제안한다. Tab 키를 누르면 수락, Esc를 누르면 거절할 수 있다.
2. **Copilot Chat**: 채팅 창에서 자연어로 질문하면 코드, 설명, 디버깅 절차 등을 답변해준다.

이 수업에서 Copilot의 역할은 **"정답 생성기"가 아니라 "코딩/디버깅/해석문 다듬기 조교"**이다. 학생이 코딩에 쓰는 시간을 줄여, 더 많은 시간을 "해석·판단"에 할당하는 것이 목표이다.

쉽게 말해서, Copilot은 "시험 대신 봐주는 친구"가 아니라, "공부할 때 옆에서 도와주는 과외 선생님"이다. 최종 답안지에 이름을 쓰는 것은 학생 본인이다.

---

####  "Copilot이 틀릴 수 있을까?"

> **질문**: "Copilot이 틀린 답을 줄 수 있다고 생각하는가?"
>
> - 투표해 보자. (예 / 아니오)

Copilot은 대규모 코드 데이터로 학습한 AI 모델이다. 따라서 **틀린 코드를 자신 있게 제안하는 경우가 있다**. 예를 들어, 함수 이름을 잘못 쓰거나, 존재하지 않는 라이브러리를 불러오거나, 논리적으로 잘못된 계산을 하는 경우가 실제로 발생한다. 이것이 바로 "검증 원칙"이 필요한 이유이다.

---

### 핵심 개념 2: 검증 원칙

**검증 원칙**이란, Copilot이 생성한 코드와 설명을 반드시 **코드 실행 결과(출력/표/그래프)**로 확인한다는 규칙이다. 검증 없이 Copilot의 답변을 그대로 제출하면 안 된다.

구체적으로, 이 수업에서 모든 과제는 다음 형식으로 제출한다. **이 형식을 앞으로 매주 사용한다.**

- **(A) 분석 질문** (1-2문장): 무엇을 알고 싶은가?
- **(B) 사용 변수/수준**: 어떤 데이터를 사용했는가?
- **(C) 결과 근거**: 핵심 표/그래프/수치
- **(D) 해석문** (5-7문장): 결과가 무엇을 의미하는가?
- **(E) Copilot 사용 요약** (2-3줄): 어떤 요청을 했고, 무엇을 검증했는가?

특히 **(E)항**이 중요하다. Copilot에게 무엇을 요청했고, 그 답변을 어떻게 검증했는지를 기록하는 것이다. "Copilot이 해줬으니 끝"이 아니라, "Copilot이 해준 것을 내가 확인했다"가 핵심이다.

---

### 핵심 개념 3: 프라이버시 규칙

**프라이버시 규칙**이란, 개인 및 민감 데이터를 Copilot에 절대 입력하지 않는다는 원칙이다.

구체적으로 다음을 Copilot Chat이나 코드에 입력하면 안 된다.

- 이름, 전화번호, 학번, 주소 등 **개인 식별 정보**
- 건강 상태, 정치 성향, 종교, 성적 등 **민감 정보**
- 비공개 자료, 타인의 과제 파일 등

이 수업에서 사용하는 데이터는 모두 **익명화된 공개 데이터 또는 수업용 가상 데이터**이다. 학생 개인의 실제 데이터를 업로드하는 것은 금지한다.

---

### Copilot 사용 정책 요약

| 구분 | 내용 |
|---|---|
| **허용** | 개념 설명 요청, 분석 절차 안내, 디버깅, 결과 해석 초안, 글쓰기 도움 |
| **금지** | 개인정보/민감정보 업로드, 시험 답안 복사 제출, 데이터 조작/위조, 타인 과제 복제 |
| **증빙** | 과제에 결과 근거 + Copilot 사용 요약 첨부 |
| **책임** | Copilot 답변은 참고이며, 최종 결과의 정확성은 **학생 책임** |

### 2단계: 기본(Basic) 트랙 실습 과제

#### ① 실행

Copilot Chat을 열고, 아래 프롬프트를 입력한다.

> "Python으로 1부터 10까지 합을 구하는 코드를 짜줘"

Copilot이 제안한 코드를 새 Python 파일에 붙여넣고 실행한다.

```python
# Copilot이 제안한 코드 (예시)
total = sum(range(1, 11))
print(total)
```

**완료 체크**: 코드를 실행하여 **55**라는 결과가 출력되었는가?

#### ② 확인

결과가 맞는지 직접 확인한다. 1+2+3+...+10 = 55이므로 결과가 올바르다.

이어서 Copilot Chat에 다음을 질문한다.

> "이 코드가 왜 이렇게 동작하는지 설명해줘"

Copilot의 설명을 읽고, **핵심 내용 1가지를 기록**한다. 예를 들어, "`range(1, 11)`은 1부터 10까지의 정수를 생성한다"와 같은 설명이 나올 수 있다.

**완료 체크**: Copilot의 설명에서 핵심 1가지를 기록했는가?

#### ③ 해석 (10분, 첫 A–E 형식 연습)

Copilot이 제안한 코드와 설명 중 **1가지를 "채택" 또는 "거절"**하고, 그 이유를 2문장으로 작성한다.

오늘은 첫 수업이므로, A–E 형식을 간소화하여 다음만 작성한다.

**제출물 구성:**

- **(A) 목표** (1문장): "Copilot을 이용하여 1부터 10까지의 합을 구하는 코드를 작성하고 검증한다."
- **(C) 결과 근거**: 실행 결과 출력 (55)
- **(D) 해석** (2문장): Copilot의 코드/설명 중 1가지를 채택 또는 거절하고, 이유를 작성한다.
- **(E) Copilot 사용 요약** (2-3줄): 어떤 질문을 했고, 어떤 답변을 받았으며, 어떻게 검증했는가?

*(예: "채택한다. Copilot이 제안한 `sum(range(1, 11))` 코드를 실행한 결과 55가 출력되었고, 1+2+...+10 = 55와 일치하므로 정확하다.")*

> **안내**: 이 A–E 형식은 앞으로 **매주 사용하는 표준 제출물 형식**이다. 다음 주부터는 (B) 사용 변수/수준까지 포함하여 완전한 형태로 작성한다.

## 정리/오류 패턴 공유 + 비판적 읽기

### 흔한 오류 3개

설치 및 실습 중 자주 나타나는 오류를 정리한다.

**오류 1: Copilot 로그인이 안 됨**

VS Code에서 Copilot 아이콘이 비활성화 상태로 남아 있거나, "Sign In" 이후에도 제안이 나오지 않는 경우이다. 해결 절차는 다음과 같다.

1. VS Code를 완전히 종료한 후 다시 실행한다.
2. Extensions에서 GitHub Copilot 확장이 "Enabled" 상태인지 확인한다.
3. 문제가 지속되면 확장을 삭제하고 재설치한다.

**오류 2: Copilot Chat 창이 보이지 않음**

코드 완성(ghost text)은 작동하지만 Chat 창이 보이지 않는 경우이다. **GitHub Copilot Chat** 확장은 GitHub Copilot과 별도의 확장이므로, Extensions에서 "GitHub Copilot Chat"을 검색하여 설치되어 있는지 확인한다.

**오류 3: 결제 화면이 등장함**

GitHub Student Developer Pack 신청 과정에서 결제 정보를 요구하는 화면이 나올 수 있다. 이 경우 **즉시 멈추고 조교/교수에게 확인**한다. 계정 상태나 학교 정책에 따라 무료 이용 가능 여부가 다르다.

### 비판적 읽기 (5–10분)

아래는 가상의 학생 제출물이다. 읽고 문제점을 찾아보자.

> **학생 X의 제출물**
>
> (A) 목표: "1부터 10까지 합을 구한다."
>
> (C) 결과: (없음)
>
> (D) 해석: "Copilot이 코드를 짜줘서 완료했다."
>
> (E) Copilot 사용: "코드를 짜달라고 했다."

> **토론 질문**: 이 제출물에서 무엇이 문제인가? (2분, 옆 사람과 토론 후 전체 공유)

**문제점 예시:**

1. **결과 근거 누락**: 코드를 실행했는지, 결과가 55인지 확인할 수 없다. "완료했다"는 말만으로는 검증이 된 것이 아니다.

2. **해석 부재**: "Copilot이 짜줘서 완료했다"는 해석이 아니다. Copilot의 제안이 왜 맞는지(또는 틀린지)를 본인의 판단으로 서술해야 한다.

3. **Copilot 사용 요약 불충분**: "코드를 짜달라고 했다"만으로는 어떤 프롬프트를 사용했고, 답변을 어떻게 검증했는지 알 수 없다. 구체적인 요청 내용과 검증 과정을 기록해야 한다.

요약하면, Copilot은 "조교"이다. 조교가 해준 일을 본인이 확인하고, 이해하고, 설명할 수 있어야 한다. 그래야 **학생 본인의 학습**이 된다.
