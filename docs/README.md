# docs/ — 강의자료 원본 (Source Material)

> **수업시간 활용 문서는 `notebooks/WNN.ipynb`로 단일화되었습니다.**

## 이 폴더의 역할

이 폴더는 강의자료의 **원본 마크다운 소스**를 보관합니다. 실제 수업에서는 이 폴더의 파일을 직접 사용하지 않고, 통합된 Jupyter 노트북을 사용합니다.

## 현재 구조

| 파일 | 통합된 위치 |
|---|---|
| `01A.md` / `01B.md` / `01C.md` | → `notebooks/W01.ipynb` |
| `02A.md` / `02B.md` / `02C.md` | → `notebooks/W02.ipynb` |
| ... | ... |
| `08A.md` / `08B.md` / `08C.md` | → `notebooks/W08.ipynb` |
| `10.md` | → `notebooks/W10.ipynb` |
| `11.md` ~ `15.md` | → `notebooks/W11.ipynb` ~ `W15.ipynb` |
| `data_pool_guide.md` | 10-15주 개인 리서치 브리프용 공공 데이터 가이드 |

## 수업 내 문서 흐름 (1-8주)

각 노트북은 회차별로 다음 순서로 구성됩니다:

1. **A회차** (강의): 핵심 개념 + 예시 + 실습 과제 안내 + 데모 코드
2. **B회차** (실습): 과제 안내 + 바이브코딩 가이드 + 과제 수행
3. **C회차** (정답): 정답지 + 흔한 오류 + 모범 보고서

## 수업 내 문서 흐름 (10-15주)

10주 이후 개인 리서치 브리프 구간은 A/B 2회차 구조로 단일 파일(`NN.md`)이며, 그대로 노트북에 통합되어 있습니다.

## 원본 수정 시 주의

`docs/*.md`를 수정했다면 `notebooks/WNN.ipynb`도 함께 업데이트해야 합니다. 통합은 수동 동기화이며 자동 빌드 파이프라인은 없습니다.

## 편집 권장 위치

- **수업 직전 수정**: `notebooks/WNN.ipynb`에서 직접 수정
- **대규모 개정 (구조 변경)**: `docs/*.md`에서 먼저 수정 후 노트북 재생성
