## A회차: 결측치 처리와 기술통계 (방법론 보강)

### 왜 결측치가 중요한가

수업 실습 데이터는 결측이 거의 없었지만, 실제 공공 마이크로데이터는 **10-30%의 결측이 일반적**이다. 결측을 무시하면:
- 표본 크기가 일관되지 않아 결과 비교가 어려워진다.
- pandas가 자동으로 결측을 제외하면서, 분석마다 N이 달라진다.
- 결측 패턴이 결과에 편향(bias)을 줄 수 있다.

---

### 결측의 3가지 유형

| 유형 | 의미 | 예시 |
|---|---|---|
| **MCAR** (완전 무작위) | 결측이 변수들과 관련 없음 | 조사원이 실수로 한 줄 빠뜨림 |
| **MAR** (조건부 무작위) | 다른 관측 변수에 의해 결측 확률이 달라짐 | 여성이 남성보다 소득 질문에 응답 거부가 많음 |
| **MNAR** (비무작위) | 결측 자체가 변수값과 관련 | 고소득자가 소득 질문에 응답 거부 |

학부 수준에서는 **MCAR 가정 하에 단순 처리**를 사용하되, 그 한계를 보고서 한계 섹션에 명시하는 것까지가 책임이다.

---

### 학부 수준 결측 처리 3가지

| 방법 | 언제 사용하는가 | 주의점 |
|---|---|---|
| **완전사례 분석 (listwise deletion)** | 결측이 5% 미만일 때 | N 감소와 잠재적 편향 명시 |
| **평균/중앙값 대체** | 연속변수, 결측이 10% 미만 | 분산이 축소됨 (보고서에 명시) |
| **"결측" 범주화** | 범주형 변수 | "무응답"이 의미를 가질 때만 |

**금지**: 다중대체(multiple imputation) 등 고급 기법은 본 과목 범위 밖이다. 사용하려면 교수와 상의한다.

---

### 공공 데이터의 결측 코드 — 반드시 변환

공공 데이터는 흔히 `-9`, `-8`, `-1`, `99`, `9999` 등을 결측 코드로 사용한다. **코드북을 먼저 확인**하고 NaN으로 변환해야 한다.

```python
import pandas as pd
import numpy as np

# 결측 코드 변환 예시
df['소득'] = df['소득'].replace([-9, -8, 9999], np.nan)
df['학력'] = df['학력'].replace([-1, 99], np.nan)

# 결측률 확인
missing_summary = pd.DataFrame({
    'N': df.count(),
    '결측': df.isna().sum(),
    '결측률(%)': (df.isna().mean() * 100).round(2)
})
print(missing_summary)
```

### 기술통계 보고 표준 형식 (표 1)

보고서의 "데이터와 방법" 섹션에 포함될 **표 1 (표본 특성)** 은 아래 형식을 따른다.

| 변수 | 척도 | N | 평균(SD) 또는 빈도(%) | 결측 N(%) |
|---|---|---|---|---|
| 연령 | 등간 | 2,354 | 45.2 (13.8) | 12 (0.5%) |
| 성별 | 명목 | 2,366 | 여 51.3% / 남 48.7% | 0 (0%) |
| 학력 | 서열 | 2,301 | 고졸 이하 32%, 대졸 이상 68% | 65 (2.7%) |
| ... | | | | |

**핵심**: N과 결측 비율을 반드시 명시한다. 이것만 잘해도 보고서 품질이 한 단계 올라간다.

10주에서 계획한 1차 검정(카이제곱/t/ANOVA 중)을 실행한다.

### 흔한 오류 3가지

| 오류 | 확인법 |
|---|---|
| 결측 코드(-9, 99 등)를 NaN으로 변환하지 않고 분석에 포함한다 | 코드북을 확인하고 `df.replace()` 또는 `na_values` 인자 사용 |
| 가정 확인 없이 검정 결과만 보고한다 | 기대빈도·정규성·등분산 중 해당 검정 가정을 먼저 확인한다 |
| p값만 보고하고 효과크기를 생략한다 | 카이제곱은 Cramer's V, t-검정은 Cohen's d, ANOVA는 η²를 함께 제시 |
