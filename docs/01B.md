## 데이터/바이브코딩 안내

### 오늘의 데이터

**social_survey_w01.csv** --- 200명의 가상 사회 설문 데이터이다. 아래 6개 변수가 포함되어 있다.

**표 W01.1** social_survey_w01.csv 변수 목록

| 변수명 | 설명 | 수준(범주) / 범위 |
|---|---|---|
| 성별 | 응답자의 성별 | 남성, 여성 |
| 연령 | 응답자의 나이 | 19--65세 |
| 교육수준 | 최종 학력 | 고졸이하, 대졸, 대학원이상 |
| 소득 | 월 소득(만원) | 연속 수치 |
| 생활만족도 | 현재 생활 만족 정도 | 1--5점 |
| 정치관심도 | 정치에 대한 관심 정도 | 1--5점 |

> **참고**: **척도(scale)**란 변수를 측정하는 수준을 말한다. **명목(nominal)**은 이름만 있는 범주(성별, 지역 등), **서열(ordinal)**은 순서가 있는 범주(만족도 1--5점), **비율(ratio)**은 절대 영점이 있는 연속 수치(연령, 소득)이다. 척도에 대해서는 다음 주(2주차)에 본격적으로 배운다. 오늘은 "변수마다 종류가 다르다"는 점만 인식하면 충분하다.

### 바이브코딩이란?

오늘 과제는 학생이 직접 코드를 작성하는 대신, **Copilot에게 데이터를 제시하고 분석을 요청**하는 방식(**바이브코딩**, vibe coding)으로 수행한다.

**바이브코딩 절차**:
1. Copilot Chat에 **구체적인 프롬프트**를 입력한다.
2. Copilot이 생성한 **코드를 복사하여 실행**한다.
3. 에러가 나면 에러 메시지를 Copilot에게 다시 보내 **수정 코드를 요청**한다.
4. 최종 코드와 실행 결과를 **모두 과제물에 포함**한다.

**바이브코딩 프롬프트 예시**:

| 용도 | 프롬프트 |
|---|---|
| 데이터 로드 + 요약 | "social_survey_w01.csv 파일을 pandas로 불러와서, 데이터 크기(행x열)와 기본 요약통계를 출력하는 코드를 짜줘." |
| 결측치 확인 | "이 데이터의 변수별 결측치 수를 확인하는 코드를 짜줘." |
| 에러 수정 | "아래 에러가 발생했다. 원인과 해결 코드를 알려줘: (에러 메시지 붙여넣기)" |

> **주의**: Copilot이 생성한 코드를 그대로 실행하기 전에, **코드가 무엇을 하는지 조원끼리 간단히 확인**한다. 코드의 의미를 전혀 모른 채 실행하는 것은 바이브코딩의 목적이 아니다.

### 과제 1: CSV 로드 + 기본 요약통계 출력

**분석 질문**: "이 데이터에는 어떤 변수가 있고, 각 변수의 기본 특성(평균, 최솟값, 최댓값 등)은 어떠한가?"

**수행 절차**:

① Copilot에게 아래 프롬프트를 입력하여 코드를 생성한다.

> "social_survey_w01.csv 파일을 pandas로 불러와서, 데이터 크기(행x열)와 기본 요약통계(describe)를 출력하는 코드를 짜줘."

② Copilot이 생성한 코드를 VS Code에 붙여넣고 실행한다. 에러가 나면 에러 메시지를 Copilot에게 보내 수정을 요청한다.

③ 실행 결과를 확인하고, 아래 기록표를 채운다.

**과제 1 결과 기록표**:

| 항목 | 기록 |
|---|---|
| 사용한 Copilot 프롬프트 | (프롬프트 전문 붙여넣기) |
| Copilot이 생성한 코드 | (코드 전문 붙여넣기) |
| 에러 발생 여부 + 수정 횟수 | 있음( ___회 수정) / 없음 |
| 행 수 x 열 수 | ___ x ___ |
| 숫자형 변수 개수 | ___ |
| 연령 평균(표준편차) | ___ (___)세 |
| 소득 평균(표준편차) | ___ (___)만원 |

> **Tip**: `df.describe()`는 기본적으로 **숫자형 변수만** 보여준다. 모든 변수(범주형 포함)의 요약을 보려면 Copilot에게 "범주형 변수 포함 요약통계도 보여줘"라고 추가 요청한다.

**완료 체크**: 기록표의 모든 칸을 채웠는가? Copilot이 생성한 코드와 실행 결과를 모두 기록했는가?

### 과제 3: 결과 종합 + A--E 보고서 작성

과제 1과 과제 2의 결과를 종합하여, A--E 형식의 보고서를 작성한다. **이것이 이번 학기 첫 번째 A--E 제출물이다.** 형식에 익숙해지는 것이 이 과제의 핵심 목표이다.

#### Copilot 검증

Copilot에게 아래 프롬프트 중 **2개를 사용**하고, 결과를 검증한다.

| 구분 | 프롬프트 |
|---|---|
| 필수 1 | "이 CSV의 각 변수가 명목/서열/등간/비율 중 어디에 해당하는지 알려줘." |
| 필수 2 | "결측치 처리 방법 3가지를 비교해줘." |
| 선택 | "이 출력 결과를 사회과학 논문 스타일로 3문장으로 요약해줘." |

Copilot의 답변 중 **1가지를 채택 또는 거절**하고, 이유를 결과 근거로 2문장 이내로 기록한다.

#### A--E 보고서 작성

조원 전원이 합의하여 A--E 형식의 보고서를 완성한다.

**조별 제출물 구성 (A--E)**:

- **(A) 분석 질문** (1--2문장): "social_survey_w01.csv 데이터의 기본 구조(변수, 척도, 결측치)는 어떠한가?"
- **(B) 사용 변수/척도**: 변수명 + 척도를 나열한다. (예: 성별(___), 연령(___), ...)
- **(C) 결과 근거**: Copilot이 생성한 **코드 전문** + **실행 결과** (`df.shape` 출력, `df.describe()` 요약, `df.isnull().sum()` 결측치 현황)
- **(D) 해석문** (5--7문장): 아래 4단계 구조를 참고하여 직접 작성한다.
  - **(분석목적)** 이 데이터의 기본 구조를 파악하기 위해 요약통계와 결측치 현황을 확인했다.
  - **(핵심근거)** 데이터는 ___행 x ___열이며, 연령 평균은 ___세, 소득 평균은 ___만원이다. 결측치는 ___변수에서 ___건 발견되었다.
  - **(결론)** 이 데이터는 ___한 특성을 가지며, 분석에 앞서 ___의 처리가 필요하다.
  - **(주의)** 이 데이터는 가상 설문이므로, 결과를 실제 모집단에 일반화할 수 없다.
- **(E) Copilot 사용 요약** (2--3줄): 바이브코딩 과정 --- 어떤 프롬프트를 사용했고, 몇 번 수정했으며, 무엇을 채택/거절했는가?

**해석문(D항) 작성 시 참고**: 오늘은 첫 번째 해석문이므로, 위 템플릿의 빈칸을 채우는 방식으로 작성해도 충분하다. 다음 주부터는 더 자유롭게 작성하되, 4단계 구조(분석목적 -- 핵심근거 -- 결론 -- 주의)는 항상 유지한다.
