# W03B. EDA 체크포인트 — 과제 3

> 주어진 데이터의 핵심 패턴을 기술통계와 그래프로 요약하고, 해석문 4단계 구조로 작성할 수 있는가?

**학습 목표**
1. Copilot(바이브코딩)을 활용하여 주요 변수의 **기술통계와 시각화**(히스토그램, 박스플롯)를 수행할 수 있다.
2. 평균과 중앙값의 차이에서 분포의 치우침을 판단하고, 이상치를 식별할 수 있다.
3. 결과를 **해석문 4단계 구조**로 작성하고, 코드와 함께 보고서로 정리하여 클래스룸에 제출할 수 있다.

> **이 과제에서 처음으로 4단계 구조(분석목적 -> 핵심근거 -> 결론 -> 주의)를 완성하여 제출한다.** 구조를 빠짐없이 갖추는 것이 핵심이다.

### 바이브코딩이란?

**바이브코딩(vibe coding)**이란 분석 목표를 자연어로 Copilot에게 설명하고, 코드를 생성받아 실행하는 방식이다. 직접 코드를 한 줄씩 작성하는 대신, "무엇을 하고 싶은지"를 명확히 전달하는 것이 핵심이다.

**바이브코딩 프롬프트 예시**:

| 순서 | 프롬프트 |
|---:|---|
| 1 | "social_survey_w03.csv를 불러와서 연령, 소득, 생활만족도의 기술통계를 출력하고, 각 변수의 히스토그램을 그려줘. 평균선(빨간 점선)과 중앙값선(초록 실선)을 함께 표시해줘." |
| 2 | "교육수준별 소득의 박스플롯을 그려줘." |
| 3 | "평균과 중앙값의 차이가 큰 변수가 있다면, 분포 형태가 어떤지 설명해줘." |

**바이브코딩 주의사항**:
- Copilot이 생성한 코드를 **반드시 실행**하여 결과를 확인한다.
- 오류가 발생하면 오류 메시지를 Copilot에게 보여주고 수정을 요청한다.
- **최종 코드와 실행 결과를 모두 과제물에 포함**한다.

### 과제 1: 수치형 변수 3개 기술통계 + 히스토그램

**분석 질문**: "연령, 소득, 생활만족도의 분포는 어떤 특성을 보이는가?"

**바이브코딩 수행 절차**:
1. Copilot에게 다음과 같이 요청한다: "social_survey_w03.csv를 불러와서 연령, 소득, 생활만족도의 기술통계(평균, 중앙값, 표준편차)를 출력하고, 각 변수의 히스토그램을 그려줘. 평균선(빨간 점선)과 중앙값선(초록 실선)을 함께 표시해줘."
2. 생성된 코드를 실행하고 결과를 확인한다.
3. 평균과 중앙값의 차이를 확인하여 분포 형태(대칭/치우침)를 판정한다.
4. 아래 결과 기록표를 채운다.

**과제 1 결과 기록표**:

**표 W03.3** 수치형 변수 기술통계 결과

| 변수 | 평균 | 중앙값 | 표준편차 | 평균-중앙값 차이 | 분포 형태 판정 |
|---|---|---|---|---|---|
| 연령 | | | | | 대칭 / 좌치우침 / 우치우침 |
| 소득 | | | | | 대칭 / 좌치우침 / 우치우침 |
| 생활만족도 | | | | | 대칭 / 좌치우침 / 우치우침 |

**판정 기준**: 평균 > 중앙값이면 우치우침(오른쪽 꼬리), 평균 < 중앙값이면 좌치우침(왼쪽 꼬리), 거의 같으면 대칭으로 판정한다.

### 과제 3: 결과 종합 + 해석문 작성

과제 1, 2의 결과를 바탕으로 A-E 형식의 해석문을 **조별로 합의**하여 작성한다. 이것이 해석문 4단계 구조의 **첫 번째 실전 제출 과제**이다.

#### ① 결과 종합

과제 1의 기술통계와 과제 2의 집단별 비교를 정리하고, "가장 눈에 띄는 패턴 2가지"를 조원끼리 합의한다.

#### ② Copilot 검증

Copilot에게 아래 프롬프트 중 **2개를 사용**하고, 결과를 검증한다.

| 구분 | 프롬프트 |
|---|---|
| 필수 1 | "이 기술통계 결과에서 평균과 중앙값의 차이가 큰 변수가 있다. 이것이 의미하는 바를 설명해줘." |
| 필수 2 | "이 박스플롯에서 이상치가 보인다. 이상치를 어떻게 처리해야 하는지 설명해줘." |
| 선택 | "이 기술통계 결과를 해석문 4단계 구조(분석목적->핵심근거->결론->주의)로 써줘." |

Copilot의 답변 중 **1가지를 채택 또는 거절**하고, 이유를 결과 근거로 2문장 이내로 기록한다.

#### ③ 종합 해석문 작성

조원 전원이 합의하여 A-E 형식의 해석문을 완성한다.

**제출물 구성 (A-E)**:

- **(A) 분석 질문** (1-2문장): "이 데이터의 응답자 특성(소득, 연령, 교육수준)은 어떤 분포를 보이는가?"
- **(B) 사용 변수/척도**: 분석한 변수명 + 척도 + 수준 (표 W03.2 참조)
- **(C) 결과 근거**: Copilot이 생성한 코드 + 기술통계표(표 W03.3) + 히스토그램/박스플롯에서 발견한 패턴 2가지
- **(D) 해석문** (5-7문장): 아래 4단계 구조를 따른다.
  - **(분석목적)** 왜 이 분석을 했는가?
  - **(핵심근거)** 어떤 수치와 패턴을 확인했는가?
  - **(결론)** 데이터에서 무엇을 말할 수 있는가?
  - **(주의)** 이 결과의 한계는 무엇인가?
- **(E) Copilot 사용 요약** (2-3줄): 어떤 질문을 했고, 어떤 답변을 받았으며, 무엇을 채택/거절했는가?

**해석문(D항) 작성 시 주의**: 조원 전원이 초안을 읽고, **4단계가 모두 포함되었는지 반드시 교차 점검**한다.

| 빠지기 쉬운 부분 | 확인 질문 |
|---|---|
| (분석목적) 생략 | "왜 이 분석을 했는지 써 있는가?" |
| (핵심근거) 수치 누락 | "평균, 중앙값, 표준편차 등 구체적 숫자가 있는가?" |
| (결론) 모호함 | "그래서 무엇을 말할 수 있는지 명확한가?" |
| (주의) 한계 누락 | "표본, 자기보고, 기술통계의 한계가 명시되었는가?" |

## 조별 발표 + 전체 토론

### 조별 발표

2-3개 조가 자신의 결과를 **1분 내외**로 발표한다. 발표 형식은 다음과 같다.

> "우리 조는 소득/연령/생활만족도의 기술통계를 분석했다. 소득은 평균 ___만 원, 중앙값 ___만 원으로 ___(대칭/치우침)이었다. 교육수준별 소득 박스플롯에서 ___가 관찰되었다."

### 전체 토론

**토론 질문**:
- "평균과 중앙값의 차이가 가장 큰 변수는 무엇이었는가? 그 이유는?"
- "박스플롯에서 이상치가 있었는가? 이상치를 제거하면 결과가 달라지는가?"
- "해석문에서 '주의' 단계를 빠뜨린 조가 있는가? 어떤 한계를 명시해야 하는가?"
