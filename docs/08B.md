### 바이브코딩이란?

오늘 실습에서는 **바이브코딩(vibe coding)** 방식으로 분석을 수행한다. 바이브코딩이란 Copilot에게 데이터와 분석 목표를 자연어로 설명하면, Copilot이 코드를 생성해 주는 방식이다. 학생은 생성된 코드를 실행하고, 결과를 확인하며, 해석문을 작성한다.

**바이브코딩 프롬프트 예시**:

| 순서 | 프롬프트 예시 |
|---:|---|
| 1 | "social_survey_w08.csv를 불러와서 소득을 종속변수, 교육연수를 독립변수로 단순회귀(OLS)를 실행하고 summary()를 출력해줘." |
| 2 | "같은 데이터에서 소득 ~ 교육연수 + 경력연수 다중회귀를 실행해줘. summary()와 VIF를 출력해줘." |
| 3 | "단순회귀 대비 다중회귀에서 교육연수 계수가 어떻게 변했는지 비교 표를 만들어줘." |
| 4 | "소득 ~ 교육연수 + 경력연수 + 성별_더미 다중회귀를 실행하고, 표준화 계수를 함께 출력해줘." |

**주의사항**:
- Copilot이 생성한 코드를 그대로 실행하기 전에, 변수명이 데이터와 일치하는지 확인한다.
- 성별 변수는 더미변수로 변환해야 한다: `df["성별_더미"] = (df["성별"] == "남성").astype(int)`
- **최종 코드와 실행 결과를 모두 과제물에 포함**한다.
- Copilot의 해석 문장은 참고만 하고, 해석문은 직접 작성한다.
- 해석문에 **"통제 ≠ 인과"를 반드시 명시**한다.

### 해석문(D항) 작성 가이드

조원 두 명이 초안을 함께 읽고, **"통제 = 인과" 오해가 있는지 반드시 교차 점검**한다.

| 잘못된 표현 (X) | 올바른 표현 (O) |
|---|---|
| "경력을 통제했으므로 교육이 원인이다" | "경력을 통계적으로 조정한 상태에서 교육과 소득의 양의 연관이 관찰된다" |
| "교육이 소득에 영향을 미친다" | "교육연수는 소득의 유의한 예측 변수이다" |
| "성별이 소득 차이를 결정한다" | "성별을 통제한 상태에서 남성의 소득이 여성보다 평균적으로 높은 것으로 추정된다" |

**해석문에 반드시 포함할 문장**: "이 결과는 ___를 의미하지 않으며(통제 ≠ 인과), 측정되지 않은 교란변수가 존재할 수 있다."
