### 바이브코딩이란?

오늘 과제는 **바이브코딩(vibe coding)** 방식으로 수행한다. 바이브코딩이란 Copilot에게 데이터와 분석 목표를 자연어로 설명하고, 코드 생성·실행을 요청하는 방식이다. 절차는 다음과 같다.

1. Copilot에게 분석 목표를 자연어로 설명한다.
2. Copilot이 생성한 코드를 읽고 실행한다.
3. 실행 결과를 확인하고, 필요하면 수정을 요청한다.
4. **최종 코드와 실행 결과를 모두 과제물에 포함**한다.

### 바이브코딩 프롬프트 예시

과제 수행 시 아래 프롬프트를 참고하여 Copilot에게 요청한다.

| 구분 | 프롬프트 |
|---|---|
| 결측 탐색 | "social_survey_w02_raw.csv를 불러와서 변수별 결측치 수와 비율을 확인해줘." |
| 이상치 탐색 | "연령 변수에서 100 이상이거나 0 미만인 이상치를 찾아줘." |

| 전후 비교 | "처리 전후의 기술통계를 비교하는 표를 만들어줘." |

### 과제 3: 정리 과정 서술 + A–E 보고서 작성

과제 1과 과제 2의 결과를 바탕으로, 정리 과정 전체를 서술하고 제출물을 완성한다. 조원 두 명이 **함께 합의**하여 작성한다.

#### ① Copilot 검증

Copilot에게 아래 프롬프트 중 **2개를 사용**하고, 결과를 검증한다.

| 구분 | 프롬프트 |
|---|---|
| 필수 1 | "이 데이터에서 이상치를 탐지하는 방법 3가지를 알려줘." |
| 필수 2 | "결측치를 중앙값으로 대체하면 분포에 어떤 영향이 있는지 설명해줘." |
| 선택 | "이 데이터의 결측 패턴이 MCAR인지 확인하는 방법을 알려줘." |

Copilot의 답변 중 **1가지를 채택 또는 거절**하고, 이유를 결과 근거로 2문장 이내로 기록한다.

#### ② 조별 합의 + A–E 보고서 작성

**조별 제출물 구성 (A–E)**:

- **(A) 분석 질문** (1–2문장): "이 데이터에서 어떤 결측/이상치 문제가 있으며, 어떻게 처리했는가?"
- **(B) 사용 변수/척도**: 각 변수의 이름, 척도(명목/서열/비율), 결측/이상치 현황
- **(C) 결과 근거**: Copilot이 생성한 코드 + 처리 전후 기술통계 비교표 (과제 2 전후 비교표)
- **(D) 해석문** (5–7문장): 아래 구조를 참고하여 직접 작성한다.
- **(E) Copilot 사용 요약** (2–3줄): 어떤 질문을 했고, 어떤 답변을 받았으며, 무엇을 채택/거절했는가?

**해석문(D항) 작성 시 참고 구조** (3주차에서 정식 도입하므로 간략 형태):

> **(분석목적)** 원시 데이터의 결측치·이상치를 확인하고, 분석 가능한 형태로 정리하기 위해 탐색적 점검을 수행했다.
>
> **(핵심근거)** ___개 변수 중 ___개에서 결측이 발견되었으며, '소득' 변수의 결측 비율이 ___% 로 가장 높았다. '연령' 변수에서 범위 밖 값(___건)이 발견되어 NaN으로 처리했다.
>
> **(결론)** 이상치 제거 후 연령의 평균이 ___에서 ___으로 변화했으며, 결측치를 중앙값으로 대체한 후 소득의 표준편차가 ___에서 ___으로 변화했다. 처리 후 남은 결측은 ___건이다.
>
> **(주의)** 중앙값 대체는 분산을 줄이는 효과가 있으므로, 실제 분석 시에는 결측 유형에 따른 적절한 처리 방법을 선택해야 한다. 특히 '소득' 변수의 결측이 ___일 가능성을 고려해야 한다.
